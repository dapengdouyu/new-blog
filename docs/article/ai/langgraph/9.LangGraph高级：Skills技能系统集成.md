# LangGraph 高级：Skills 技能系统集成

在前面的教程中，我们把 Agent 的所有能力都写在代码里——工具定义在函数中，指令写在系统提示词中。但当 Agent 需要掌握的"专业知识"越来越多时，全部塞进提示词会导致 Token 爆炸，而散落在代码各处又难以维护。

**Skills（技能）** 是一种将领域知识和工作流程模块化封装的方案。本篇将教你如何在纯 LangGraph 图中实现技能加载机制。

## 什么是 Skills？

Skills 遵循 [Agent Skills 规范](https://agentskills.io/specification)，每个技能是一个目录，包含一个 `SKILL.md` 文件：

```
skills/
├── code-review/
│   ├── SKILL.md           # 技能定义（必须）
│   └── lint_check.py      # 辅助脚本（可选）
├── data-analysis/
│   └── SKILL.md
└── web-research/
    └── SKILL.md
```

### SKILL.md 的结构

```markdown
---
name: code-review
description: 对代码变更进行结构化审查，检查正确性、风格、测试和潜在问题。
---

# 代码审查技能

## 使用时机
- 完成代码修改后需要验证质量
- 用户要求审查代码

## 审查步骤
1. 完整阅读每个修改的文件
2. 运行测试套件
3. 对照检查清单逐项验证
...
```

YAML frontmatter 提供元数据（名称、描述），正文包含详细指令。

## 为什么不把所有技能都放进提示词？

假设你有 5 个技能，每个 SKILL.md 约 200 行。如果全部放进系统提示词：

```
5 × 200 行 ≈ 1000 行 ≈ ~4000 tokens → 每次调用都消耗
```

而大部分情况下，一次任务只会用到 1-2 个技能。解决方案是 **渐进式披露（Progressive Disclosure）**：

```
系统提示词中只放摘要：
  - code-review: 对代码变更进行结构化审查（20 tokens）
  - data-analysis: 使用 pandas 进行数据分析（20 tokens）
  - web-research: 结构化的网络研究流程（20 tokens）

Agent 判断需要时 → 调用 read_skill 读取完整内容 → 按指令执行
```

## 实现步骤

### 第一步：扫描技能目录

启动时读取所有 SKILL.md 的 frontmatter，提取名称和描述：

```python
import os
import re
import yaml


def load_skills_metadata(skills_dir: str) -> dict[str, dict]:
    """扫描 skills 目录，解析每个 SKILL.md 的 YAML frontmatter。"""
    skills = {}
    if not os.path.isdir(skills_dir):
        return skills

    for entry in os.listdir(skills_dir):
        skill_file = os.path.join(skills_dir, entry, "SKILL.md")
        if not os.path.isfile(skill_file):
            continue

        with open(skill_file, encoding="utf-8") as f:
            content = f.read()

        # 解析 YAML frontmatter（--- 分隔）
        match = re.match(r"^---\s*\n(.*?)\n---", content, re.DOTALL)
        if not match:
            continue

        meta = yaml.safe_load(match.group(1))
        if not meta or "name" not in meta:
            continue

        skills[meta["name"]] = {
            "description": meta.get("description", ""),
            "path": skill_file,
        }

    return skills
```

### 第二步：构建系统提示词

将技能摘要注入系统提示词，并告诉 Agent 如何使用：

```python
def build_system_prompt(base_prompt: str, skills: dict[str, dict]) -> str:
    """在基础提示词后追加技能列表。"""
    if not skills:
        return base_prompt

    skills_section = "\n\n## 可用技能\n\n"
    skills_section += "当任务匹配某个技能时，先用 `read_skill` 工具读取完整指令，再按指令执行。\n\n"

    for name, meta in skills.items():
        skills_section += f"- **{name}**：{meta['description']}（路径：`{meta['path']}`）\n"

    return base_prompt + skills_section
```

### 第三步：提供 read_skill 工具

Agent 判断需要某个技能时，通过这个工具读取完整内容：

```python
from langchain_core.tools import tool

# 全局技能注册表（启动时加载一次）
SKILLS_REGISTRY: dict[str, dict] = {}


@tool
def read_skill(skill_name: str) -> str:
    """读取指定技能的完整 SKILL.md 内容。当你判断某个技能适用于当前任务时调用此工具。

    Args:
        skill_name: 技能名称，如 "code-review"、"data-analysis"
    """
    meta = SKILLS_REGISTRY.get(skill_name)
    if not meta:
        available = ", ".join(SKILLS_REGISTRY.keys()) or "无"
        return f"技能 '{skill_name}' 不存在。可用技能：{available}"

    try:
        with open(meta["path"], encoding="utf-8") as f:
            return f.read()
    except OSError as e:
        return f"读取技能文件失败：{e}"
```

### 第四步：组装 LangGraph 图

将以上组件组合成一个完整的 LangGraph Agent：

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langchain_openai import ChatOpenAI
from langchain_core.messages import SystemMessage
from dotenv import load_dotenv

load_dotenv()


# ---------- 状态定义 ----------

class State(TypedDict):
    messages: Annotated[list, add_messages]


# ---------- 加载技能 ----------

SKILLS_DIR = "./skills"
SKILLS_REGISTRY.update(load_skills_metadata(SKILLS_DIR))

BASE_PROMPT = "你是一个智能助手，能够根据任务需要调用技能来完成复杂工作。"
system_prompt = build_system_prompt(BASE_PROMPT, SKILLS_REGISTRY)


# ---------- 定义业务工具（可选） ----------

@tool
def execute_python(code: str) -> str:
    """在沙箱中执行 Python 代码并返回输出。

    Args:
        code: 要执行的 Python 代码
    """
    import subprocess
    result = subprocess.run(
        ["python3", "-c", code],
        capture_output=True, text=True, timeout=30,
    )
    output = result.stdout
    if result.returncode != 0:
        output += f"\n[错误] {result.stderr}"
    return output or "(无输出)"


# ---------- 绑定工具到 LLM ----------

all_tools = [read_skill, execute_python]
tool_map = {t.name: t for t in all_tools}
llm = ChatOpenAI(model="gpt-4o").bind_tools(all_tools)


# ---------- 图节点 ----------

def agent_node(state: State):
    """Agent 节点：调用 LLM 决策。"""
    messages = [SystemMessage(content=system_prompt)] + state["messages"]
    response = llm.invoke(messages)
    return {"messages": [response]}


def tool_node(state: State):
    """工具节点：执行 Agent 请求的工具调用。"""
    last_message = state["messages"][-1]
    results = []
    for call in last_message.tool_calls:
        func = tool_map.get(call["name"])
        if func:
            result = func.invoke(call["args"])
        else:
            result = f"未知工具：{call['name']}"
        results.append({
            "role": "tool",
            "content": str(result),
            "tool_call_id": call["id"],
        })
    return {"messages": results}


def should_continue(state: State):
    """判断是否需要继续调用工具。"""
    last = state["messages"][-1]
    if getattr(last, "tool_calls", None):
        return "tools"
    return END


# ---------- 构建图 ----------

graph_builder = StateGraph(State)
graph_builder.add_node("agent", agent_node)
graph_builder.add_node("tools", tool_node)

graph_builder.add_edge(START, "agent")
graph_builder.add_conditional_edges("agent", should_continue, {"tools": "tools", END: END})
graph_builder.add_edge("tools", "agent")

graph = graph_builder.compile()
```

### 执行流程

```
[用户] "分析 data.csv 的销售趋势"
   │
   ▼
[Agent] 看到系统提示词中有 data-analysis 技能
   │
   ├── 调用 read_skill("data-analysis")
   │      └── 返回 SKILL.md 完整内容（分析流程、注意事项…）
   │
   ├── 按 SKILL.md 指令执行：
   │      ├── execute_python("import pandas as pd; ...")
   │      ├── execute_python("df.describe()")
   │      └── execute_python("df.plot(...)")
   │
   └── 返回分析报告
```

## 完整示例：创建技能并运行

### 1. 创建技能文件

```bash
mkdir -p skills/data-analysis
```

`skills/data-analysis/SKILL.md`：

```markdown
---
name: data-analysis
description: 使用 Python 和 pandas 分析数据文件，生成统计摘要和洞察。当用户要求分析 CSV、Excel 或数据报告时使用。
---

# 数据分析技能

## 分析流程

### 第一步：数据探索
1. 读取数据文件，查看前 5 行和列信息
2. 确认数据类型、行数、列数

### 第二步：数据质量检查
1. 检查缺失值比例
2. 检查异常值（超出 3 倍标准差）
3. 记录数据质量问题

### 第三步：统计分析
1. 计算描述性统计（均值、中位数、标准差）
2. 按关键维度分组聚合
3. 计算相关性（如适用）

### 第四步：输出报告
- 用表格呈现关键指标
- 列出发现的 3-5 个核心洞察
- 给出数据驱动的建议

## 注意事项
- 大文件（>10万行）先采样分析
- 数值结果保留 2 位小数
- 始终说明数据的局限性
```

### 2. 运行 Agent

```python
result = graph.invoke({
    "messages": [{"role": "user", "content": "分析 data.csv 中各月的销售额趋势"}]
})

print(result["messages"][-1].content)
```

Agent 会先调用 `read_skill("data-analysis")` 获取完整指令，然后按照"数据探索 → 质量检查 → 统计分析 → 输出报告"的流程执行。

## 进阶：多层技能优先级

可以从多个目录加载技能，后加载的覆盖先加载的：

```python
def load_skills_from_sources(sources: list[str]) -> dict[str, dict]:
    """从多个来源加载技能，后者覆盖前者。"""
    merged = {}
    for source in sources:
        merged.update(load_skills_metadata(source))
    return merged

# 基础技能 → 用户自定义 → 项目特定（优先级递增）
SKILLS_REGISTRY.update(
    load_skills_from_sources([
        "./skills/base/",
        "./skills/user/",
        "./skills/project/",
    ])
)
```

这样你可以在组织级别提供通用技能，在项目级别覆盖或添加特定技能。

## 进阶：使用 LangGraph ToolNode 简化

前面我们手动实现了 `tool_node`。LangGraph 的 `ToolNode` 可以简化这部分：

```python
from langgraph.prebuilt import ToolNode

all_tools = [read_skill, execute_python]
llm = ChatOpenAI(model="gpt-4o").bind_tools(all_tools)

graph_builder = StateGraph(State)
graph_builder.add_node("agent", agent_node)
graph_builder.add_node("tools", ToolNode(all_tools))  # 自动处理工具调用

graph_builder.add_edge(START, "agent")
graph_builder.add_conditional_edges("agent", should_continue, {"tools": "tools", END: END})
graph_builder.add_edge("tools", "agent")

graph = graph_builder.compile()
```

## 与 Deep Agents 的 SkillsMiddleware 对比

Deep Agents 框架将本篇的逻辑封装为 `SkillsMiddleware`，自动完成技能扫描、提示词注入和按需加载：

```python
# Deep Agents 方式（一行搞定）
from deepagents import create_deep_agent

agent = create_deep_agent(
    model="openai:gpt-4o",
    skills=["/skills/user/", "/skills/project/"],
)
```

对比：

| 方面 | 纯 LangGraph（本篇） | Deep Agents |
|------|---------------------|-------------|
| 技能扫描 | 手动实现 `load_skills_metadata` | `SkillsMiddleware` 自动处理 |
| 提示词注入 | 手动拼接到 system prompt | Middleware 自动注入 |
| 按需读取 | 手动定义 `read_skill` 工具 | 复用内置 `read_file` 工具 |
| 多来源优先级 | 手动合并 | 自动 last-one-wins |
| 适用场景 | 需要完全控制图结构 | 快速构建、开箱即用 |

::: tip 如何选择
如果你的项目已经在使用 LangGraph 构建自定义图，并且需要精细控制节点逻辑，用本篇的方式手动集成更灵活。如果你想快速获得"深度 Agent"的全部能力（规划、文件系统、子代理、技能），直接使用 [Deep Agents](../deepagents/) 更高效。
:::

## 核心概念回顾

| 概念 | 说明 |
|------|------|
| SKILL.md | 技能定义文件，包含 YAML frontmatter 和 Markdown 指令 |
| YAML frontmatter | `---` 分隔的元数据块，定义 name 和 description |
| 渐进式披露 | 系统提示词只放摘要，Agent 按需读取完整内容 |
| `read_skill` 工具 | Agent 用来按需加载技能完整内容的工具 |
| 多来源优先级 | 从多个目录加载技能，后加载的覆盖先加载的 |
| Agent Skills 规范 | Skills 的标准化定义规范 |
