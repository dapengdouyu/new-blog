# LangGraph 进阶：持久化与检查点

之前的示例中，每次调用 `graph.invoke()` 都是独立的——没有对话记忆。本篇将介绍 LangGraph 的**检查点（Checkpointer）**机制，让应用具备状态持久化和多轮对话能力。

## 为什么需要持久化？

- **多轮对话**：记住之前的对话内容
- **断点恢复**：长时间运行的任务中断后可以继续
- **时间回溯**：查看或回退到任意历史状态
- **多线程**：同一个图支持多个独立的对话线程

## Checkpointer 基础

### MemorySaver（内存存储）

最简单的检查点实现，数据存储在内存中：

```python
from langgraph.checkpoint.memory import MemorySaver

memory = MemorySaver()
graph = graph_builder.compile(checkpointer=memory)
```

### 使用 thread_id 管理对话

通过 `thread_id` 区分不同的对话会话：

```python
config = {"configurable": {"thread_id": "user-001"}}

# 第一轮对话
result1 = graph.invoke(
    {"messages": [{"role": "user", "content": "我叫小明"}]},
    config
)
# AI: 你好小明！

# 第二轮对话（同一个 thread_id）
result2 = graph.invoke(
    {"messages": [{"role": "user", "content": "我叫什么名字？"}]},
    config
)
# AI: 你叫小明。  ← 记住了上下文
```

不同的 `thread_id` 有独立的上下文：

```python
config_b = {"configurable": {"thread_id": "user-002"}}

result = graph.invoke(
    {"messages": [{"role": "user", "content": "我叫什么名字？"}]},
    config_b
)
# AI: 你还没告诉我你的名字。  ← 独立的对话线程
```

## 完整示例：带记忆的聊天机器人

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langgraph.checkpoint.memory import MemorySaver
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv

load_dotenv()

class State(TypedDict):
    messages: Annotated[list, add_messages]

llm = ChatOpenAI(model="gpt-4o-mini")

def chatbot(state: State):
    response = llm.invoke(state["messages"])
    return {"messages": [response]}

graph_builder = StateGraph(State)
graph_builder.add_node("chatbot", chatbot)
graph_builder.add_edge(START, "chatbot")
graph_builder.add_edge("chatbot", END)

memory = MemorySaver()
graph = graph_builder.compile(checkpointer=memory)

def chat():
    config = {"configurable": {"thread_id": "demo"}}
    print("带记忆的聊天机器人（输入 quit 退出）")

    while True:
        user_input = input("\n你: ")
        if user_input.lower() == "quit":
            break

        result = graph.invoke(
            {"messages": [{"role": "user", "content": user_input}]},
            config
        )
        print(f"AI: {result['messages'][-1].content}")

if __name__ == "__main__":
    chat()
```

## 检查点的工作原理

每次节点执行后，LangGraph 都会自动保存一个**检查点快照**：

```
invoke(msg1)
  ├─ checkpoint 0: 初始状态
  ├─ [chatbot] 执行
  └─ checkpoint 1: 节点执行后状态

invoke(msg2)
  ├─ checkpoint 1: 恢复上次状态（包含 msg1 的对话历史）
  ├─ [chatbot] 执行
  └─ checkpoint 2: 节点执行后状态
```

### 查看状态历史

```python
config = {"configurable": {"thread_id": "demo"}}

# 获取当前状态
current_state = graph.get_state(config)
print(f"当前消息数: {len(current_state.values['messages'])}")

# 获取状态历史
for state in graph.get_state_history(config):
    print(f"检查点: {state.config['configurable']['checkpoint_id']}")
    print(f"  消息数: {len(state.values.get('messages', []))}")
```

### 回退到历史状态

```python
# 获取历史状态
history = list(graph.get_state_history(config))

# 选择要回退到的检查点
old_state = history[2]  # 第 3 个检查点

# 从该检查点继续执行
result = graph.invoke(
    {"messages": [{"role": "user", "content": "继续之前的对话"}]},
    old_state.config  # 使用历史检查点的 config
)
```

## 持久化存储

`MemorySaver` 的数据只在进程内存中，重启就丢失了。生产环境应使用持久化存储。

### SQLite 存储

```python
from langgraph.checkpoint.sqlite import SqliteSaver

with SqliteSaver.from_conn_string("checkpoints.db") as memory:
    graph = graph_builder.compile(checkpointer=memory)
    # 数据持久化到 SQLite 文件
```

### PostgreSQL 存储

```python
from langgraph.checkpoint.postgres import PostgresSaver

DB_URI = "postgresql://user:pass@localhost:5432/langgraph"

with PostgresSaver.from_conn_string(DB_URI) as memory:
    memory.setup()  # 首次使用时创建表
    graph = graph_builder.compile(checkpointer=memory)
```

::: tip 选择存储方案
| 方案 | 适用场景 |
|------|----------|
| `MemorySaver` | 开发、测试、原型验证 |
| `SqliteSaver` | 单机部署、中小规模应用 |
| `PostgresSaver` | 生产环境、多实例部署 |
:::

## 状态更新与手动修改

你可以在图执行之外手动修改状态：

```python
config = {"configurable": {"thread_id": "demo"}}

# 手动更新状态
graph.update_state(
    config,
    {"messages": [{"role": "user", "content": "（系统注入：请用英文回答）"}]}
)

# 后续执行会包含手动注入的消息
result = graph.invoke(
    {"messages": [{"role": "user", "content": "你好"}]},
    config
)
```

这在以下场景中很有用：
- 注入系统消息
- 修正错误的中间状态
- 人机协作中的人工干预

## 带记忆的 Agent

将检查点和工具调用结合：

```python
from langgraph.prebuilt import create_react_agent
from langgraph.checkpoint.memory import MemorySaver
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool

@tool
def get_user_info(user_id: str) -> str:
    """根据用户ID查询用户信息"""
    users = {"001": "小明，25岁，工程师", "002": "小红，30岁，设计师"}
    return users.get(user_id, "未找到用户")

llm = ChatOpenAI(model="gpt-4o-mini")
memory = MemorySaver()

graph = create_react_agent(
    llm,
    tools=[get_user_info],
    checkpointer=memory,
    prompt="你是一个用户管理助手，请用中文回答。记住对话中的信息。"
)

config = {"configurable": {"thread_id": "session-1"}}

# 多轮对话
graph.invoke({"messages": [{"role": "user", "content": "查一下001号用户"}]}, config)
# AI: 001号用户是小明，25岁，工程师。

graph.invoke({"messages": [{"role": "user", "content": "他多大了？"}]}, config)
# AI: 小明今年25岁。  ← 记住了上文
```

## 小结

| 概念 | 说明 |
|------|------|
| `MemorySaver` | 内存检查点，适合开发测试 |
| `thread_id` | 区分不同对话会话的唯一标识 |
| `get_state()` | 获取当前状态快照 |
| `get_state_history()` | 获取完整的状态历史 |
| `update_state()` | 手动修改状态 |
| `SqliteSaver` / `PostgresSaver` | 持久化存储方案 |
