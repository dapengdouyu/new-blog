# LangGraph 高级：流式输出与生产部署

前面的教程都使用 `invoke()` 同步获取结果。但在实际应用中，用户更期望**实时看到输出**。本篇将介绍流式输出、错误处理以及生产部署方案。

## 流式输出

### stream() 基础

使用 `stream()` 替代 `invoke()`，逐步获取每个节点的输出：

```python
for event in graph.stream(
    {"messages": [{"role": "user", "content": "你好"}]},
    config
):
    print(event)
```

每个 `event` 是一个字典，key 为节点名，value 为该节点的输出：

```python
{"agent": {"messages": [AIMessage(content="你好！有什么...")]}}
{"tools": {"messages": [ToolMessage(content="...")]}}
{"agent": {"messages": [AIMessage(content="根据搜索结果...")]}}
```

### 流式模式

LangGraph 支持多种流式模式：

```python
# 默认：输出每个节点的完整结果
for event in graph.stream(input, config, stream_mode="values"):
    messages = event.get("messages", [])
    if messages:
        print(messages[-1].content)

# updates 模式：只输出状态变更
for event in graph.stream(input, config, stream_mode="updates"):
    for node_name, update in event.items():
        print(f"[{node_name}]: {update}")
```

| 模式 | 说明 |
|------|------|
| `"values"` | 每步输出完整状态 |
| `"updates"` | 每步只输出变更部分 |
| `"messages"` | 流式输出 LLM 生成的 token |

### Token 级流式输出

最细粒度的流式，实时输出每个 token：

```python
async for event in graph.astream_events(
    {"messages": [{"role": "user", "content": "请写一首诗"}]},
    config,
    version="v2"
):
    if event["event"] == "on_chat_model_stream":
        token = event["data"]["chunk"].content
        if token:
            print(token, end="", flush=True)
```

### 结合 FastAPI 实现 SSE

```python
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import json

app = FastAPI()

@app.post("/chat")
async def chat(request: dict):
    async def event_generator():
        async for event in graph.astream_events(
            {"messages": [{"role": "user", "content": request["message"]}]},
            {"configurable": {"thread_id": request.get("thread_id", "default")}},
            version="v2"
        ):
            if event["event"] == "on_chat_model_stream":
                token = event["data"]["chunk"].content
                if token:
                    yield f"data: {json.dumps({'token': token})}\n\n"
        yield "data: [DONE]\n\n"

    return StreamingResponse(event_generator(), media_type="text/event-stream")
```

## 错误处理

### 节点级重试

```python
from langgraph.graph import StateGraph

graph_builder = StateGraph(State)

graph_builder.add_node(
    "llm_call",
    llm_node,
    retry={"max_attempts": 3, "delay": 1.0, "backoff_factor": 2.0}
)
```

### 自定义错误处理节点

```python
def safe_tool_call(state: State):
    try:
        result = risky_operation(state)
        return {"result": result, "error": None}
    except Exception as e:
        return {"result": None, "error": str(e)}

def route_after_tool(state: State) -> str:
    if state.get("error"):
        return "error_handler"
    return "next_step"

graph_builder.add_node("tool_call", safe_tool_call)
graph_builder.add_node("error_handler", handle_error)
graph_builder.add_conditional_edges("tool_call", route_after_tool)
```

### 全局异常处理

```python
try:
    result = graph.invoke(input, config)
except GraphRecursionError:
    print("Agent 陷入循环，已达到最大递归次数")
except Exception as e:
    print(f"执行出错: {e}")
    # 可以从最近的检查点恢复
    state = graph.get_state(config)
    print(f"最后状态: {state.values}")
```

## 性能优化

### 1. 控制消息历史长度

对话越长，Token 消耗越大。定期裁剪历史消息：

```python
from langchain_core.messages import trim_messages

def agent_with_trim(state: State):
    trimmed = trim_messages(
        state["messages"],
        max_tokens=4000,
        strategy="last",
        token_counter=llm,
    )
    response = llm.invoke(trimmed)
    return {"messages": [response]}
```

### 2. 消息摘要压缩

```python
def summarize_history(state: State):
    if len(state["messages"]) > 20:
        summary_prompt = "请用 2-3 句话总结以下对话的关键信息：\n"
        summary_prompt += "\n".join(
            f"{m.type}: {m.content}" for m in state["messages"][:-5]
        )
        summary = llm.invoke(summary_prompt)

        from langchain_core.messages import SystemMessage
        return {
            "messages": [
                SystemMessage(content=f"之前的对话摘要：{summary.content}"),
                *state["messages"][-5:]
            ]
        }
    return {}
```

### 3. 并行节点

利用 LangGraph 的自动并行能力：

```python
# 这两个节点会自动并行执行
graph_builder.add_edge("input", "fetch_a")
graph_builder.add_edge("input", "fetch_b")
graph_builder.add_edge("fetch_a", "merge")
graph_builder.add_edge("fetch_b", "merge")
```

### 4. 异步执行

使用异步可以显著提升 I/O 密集型任务的性能：

```python
async def async_agent(state: State):
    response = await llm.ainvoke(state["messages"])
    return {"messages": [response]}

# 异步调用
result = await graph.ainvoke(input, config)

# 异步流式
async for event in graph.astream(input, config):
    process(event)
```

## LangGraph Platform

LangGraph Platform 是 LangChain 提供的托管部署方案。

### 本地部署（LangGraph CLI）

```bash
pip install langgraph-cli

# 创建配置文件 langgraph.json
```

`langgraph.json`：

```json
{
  "dependencies": ["."],
  "graphs": {
    "my_agent": "./agent.py:graph"
  },
  "env": ".env"
}
```

```bash
# 启动本地服务
langgraph dev
```

### 使用 LangGraph SDK 调用

```python
from langgraph_sdk import get_client

client = get_client(url="http://localhost:8123")

# 创建线程
thread = await client.threads.create()

# 流式调用
async for event in client.runs.stream(
    thread["thread_id"],
    "my_agent",
    input={"messages": [{"role": "user", "content": "你好"}]}
):
    print(event)
```

### Docker 部署

```bash
langgraph build -t my-agent-image
docker run -p 8123:8000 my-agent-image
```

### LangGraph Cloud

托管在 LangSmith 平台上，提供：
- 自动扩缩容
- 监控和追踪
- 版本管理
- Cron 定时任务

## 可观测性

### LangSmith 集成

```python
import os

os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_API_KEY"] = "your-api-key"
os.environ["LANGCHAIN_PROJECT"] = "my-langgraph-app"
```

配置后，所有的图执行、LLM 调用、工具调用都会自动记录到 LangSmith 中，方便调试和监控。

### 自定义日志

```python
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger("my_agent")

def agent_node(state: State):
    logger.info(f"Agent 收到 {len(state['messages'])} 条消息")
    response = llm.invoke(state["messages"])
    logger.info(f"Agent 回复: {response.content[:100]}...")
    return {"messages": [response]}
```
