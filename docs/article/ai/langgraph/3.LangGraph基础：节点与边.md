# LangGraph 基础：节点与边

节点和边是图的两大基本元素。本篇将深入讲解各种类型的节点和边，以及如何用它们构建灵活的工作流。

## 节点（Node）

### 节点的基本形式

节点就是一个接收状态、返回状态更新的函数：

```python
def my_node(state: State) -> dict:
    # 处理逻辑
    return {"field": "new_value"}
```

**规则**：
- 参数：接收当前完整状态
- 返回：一个字典，包含要更新的字段
- 返回 `None` 或 `{}` 表示不更新状态

### 添加节点

```python
graph_builder.add_node("node_name", my_node)
```

节点名称在图中必须唯一，它被用于定义边和调试。

::: tip 简写
如果不指定名称，LangGraph 会用函数名作为节点名称：
```python
graph_builder.add_node(my_node)  # 节点名 = "my_node"
```
:::

### 异步节点

LangGraph 完整支持异步：

```python
async def async_node(state: State):
    response = await llm.ainvoke(state["messages"])
    return {"messages": [response]}

# 使用异步调用
result = await graph.ainvoke({"messages": [...]})
```

## 边（Edge）

边定义了节点之间的执行顺序。LangGraph 提供三种类型的边。

### 1. 普通边

最简单的边，从节点 A 无条件跳转到节点 B：

```python
graph_builder.add_edge("node_a", "node_b")
```

```
[node_a] → [node_b]
```

### 2. 条件边

根据运行时状态动态选择下一个节点：

```python
def route_function(state: State) -> str:
    if state["need_search"]:
        return "search"
    return "generate"

graph_builder.add_conditional_edges(
    "classifier",         # 源节点
    route_function,       # 路由函数
)
```

路由函数返回目标节点的名称。LangGraph 会根据返回值跳转到对应节点。

```
               ┌─→ [search]
[classifier] ──┤
               └─→ [generate]
```

#### 使用映射重命名

当路由函数的返回值和节点名称不一致时，可以使用路径映射：

```python
def route_function(state: State) -> str:
    if state["is_valid"]:
        return "valid"
    return "invalid"

graph_builder.add_conditional_edges(
    "validator",
    route_function,
    {
        "valid": "process",       # "valid" → 跳转到 process 节点
        "invalid": "error_handler" # "invalid" → 跳转到 error_handler 节点
    }
)
```

### 3. 入口条件边

从 `START` 开始的条件路由，决定图的第一个执行节点：

```python
def entry_route(state: State) -> str:
    if state.get("cached_answer"):
        return "return_cached"
    return "process"

graph_builder.add_conditional_edges(START, entry_route)
```

## 实战：构建分类路由系统

一个根据用户问题类型自动路由的系统：

```python
from typing import Annotated, Literal
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv

load_dotenv()

class State(TypedDict):
    messages: Annotated[list, add_messages]
    category: str

llm = ChatOpenAI(model="gpt-4o-mini")

def classify(state: State):
    """对用户问题进行分类"""
    last_message = state["messages"][-1].content
    response = llm.invoke(
        f"将以下问题分类为 'technical' 或 'general'，只返回分类标签：\n{last_message}"
    )
    return {"category": response.content.strip().lower()}

def handle_technical(state: State):
    """处理技术问题"""
    response = llm.invoke([
        {"role": "system", "content": "你是一个技术专家，请详细回答技术问题，必要时提供代码示例。"},
        *state["messages"]
    ])
    return {"messages": [response]}

def handle_general(state: State):
    """处理通用问题"""
    response = llm.invoke([
        {"role": "system", "content": "你是一个友好的助手，请简洁明了地回答问题。"},
        *state["messages"]
    ])
    return {"messages": [response]}

def route_by_category(state: State) -> str:
    if "technical" in state["category"]:
        return "handle_technical"
    return "handle_general"

# 构建图
graph_builder = StateGraph(State)

graph_builder.add_node("classify", classify)
graph_builder.add_node("handle_technical", handle_technical)
graph_builder.add_node("handle_general", handle_general)

graph_builder.add_edge(START, "classify")
graph_builder.add_conditional_edges("classify", route_by_category)
graph_builder.add_edge("handle_technical", END)
graph_builder.add_edge("handle_general", END)

graph = graph_builder.compile()
```

执行结果：

```python
# 技术问题
result = graph.invoke({
    "messages": [{"role": "user", "content": "Python 的 GIL 是什么？"}]
})
# → 被路由到 handle_technical

# 通用问题
result = graph.invoke({
    "messages": [{"role": "user", "content": "今天天气怎么样？"}]
})
# → 被路由到 handle_general
```

## 图中的循环

LangGraph 支持循环——节点可以跳转回之前的节点。这是构建 Agent 的关键能力：

```python
def should_continue(state: State) -> str:
    last_message = state["messages"][-1]
    if hasattr(last_message, "tool_calls") and last_message.tool_calls:
        return "tools"
    return END

graph_builder.add_conditional_edges("agent", should_continue)
graph_builder.add_edge("tools", "agent")  # 工具执行后回到 agent
```

```
              ┌──────────────┐
              ↓              │
[START] → [agent] → [tools] ─┘
              │
              ↓
            [END]
```

Agent 节点判断是否需要调用工具：
- 需要 → 跳转到 tools 节点 → 执行工具 → 回到 agent
- 不需要 → 结束

这个循环会持续直到 Agent 决定不再调用工具。

::: warning 防止无限循环
LangGraph 提供了 `recursion_limit` 参数控制最大循环次数：
```python
result = graph.invoke(input, {"recursion_limit": 25})
```
默认值为 25，超过后抛出 `GraphRecursionError`。
:::

## 并行执行

当多个节点没有依赖关系时，可以并行执行：

```python
graph_builder.add_edge(START, "fetch_weather")
graph_builder.add_edge(START, "fetch_news")
graph_builder.add_edge("fetch_weather", "summarize")
graph_builder.add_edge("fetch_news", "summarize")
```

```
           ┌─→ [fetch_weather] ─┐
[START] ──┤                     ├─→ [summarize] → [END]
           └─→ [fetch_news]   ─┘
```

`fetch_weather` 和 `fetch_news` 会被**并行执行**，两者都完成后才会执行 `summarize`。

## 小结

| 概念 | 说明 |
|------|------|
| 普通边 | `add_edge(a, b)` — 无条件跳转 |
| 条件边 | `add_conditional_edges(src, fn)` — 按路由函数动态跳转 |
| 循环 | 边可以指向之前的节点，构成循环 |
| 并行 | 从同一节点出发的多条边会触发并行执行 |
| `recursion_limit` | 防止无限循环的安全阀 |
