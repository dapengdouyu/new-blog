# LangGraph 高级：子图与多 Agent 系统

当应用变得复杂时，把所有逻辑放在一个图中会难以维护。LangGraph 支持将图**嵌套组合**，以及让多个 Agent 协作完成任务。

## 子图（Subgraph）

子图就是把一个编译好的图作为另一个图的节点使用。

### 基本用法

```python
from langgraph.graph import StateGraph, START, END

# 子图：文档处理流程
class DocState(TypedDict):
    content: str
    summary: str

def extract(state: DocState):
    return {"content": f"提取的内容：{state['content'][:100]}"}

def summarize(state: DocState):
    return {"summary": f"摘要：{state['content'][:50]}..."}

doc_builder = StateGraph(DocState)
doc_builder.add_node("extract", extract)
doc_builder.add_node("summarize", summarize)
doc_builder.add_edge(START, "extract")
doc_builder.add_edge("extract", "summarize")
doc_builder.add_edge("summarize", END)

doc_graph = doc_builder.compile()
```

### 将子图嵌入主图

```python
class MainState(TypedDict):
    content: str
    summary: str
    final_report: str

def generate_report(state: MainState):
    return {"final_report": f"报告基于: {state['summary']}"}

main_builder = StateGraph(MainState)

# 将子图作为节点添加
main_builder.add_node("process_doc", doc_graph)
main_builder.add_node("report", generate_report)

main_builder.add_edge(START, "process_doc")
main_builder.add_edge("process_doc", "report")
main_builder.add_edge("report", END)

main_graph = main_builder.compile()
```

::: tip 状态传递
当子图的状态类型与主图**共享相同字段**时，LangGraph 自动传递匹配的字段。不匹配的字段在子图内独立管理。
:::

### 使用状态转换函数

当子图和主图的状态结构不同时，使用转换函数适配：

```python
class MainState(TypedDict):
    raw_text: str
    processed: str

class SubState(TypedDict):
    input_text: str
    output_text: str

def enter_sub(state: MainState) -> SubState:
    return {"input_text": state["raw_text"]}

def exit_sub(state: SubState) -> dict:
    return {"processed": state["output_text"]}

# 用函数包装子图
def sub_node(state: MainState):
    sub_input = enter_sub(state)
    sub_result = sub_graph.invoke(sub_input)
    return exit_sub(sub_result)

main_builder.add_node("sub_process", sub_node)
```

## 多 Agent 架构

### 模式一：Supervisor（主管模式）

一个 Supervisor Agent 协调多个专业 Agent：

```
                    ┌→ [研究员 Agent]
[用户] → [Supervisor] ─┼→ [写作 Agent]
                    └→ [代码 Agent]
```

```python
from typing import Annotated, Literal
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langgraph.prebuilt import create_react_agent
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.messages import HumanMessage
from dotenv import load_dotenv

load_dotenv()

class State(TypedDict):
    messages: Annotated[list, add_messages]
    next_agent: str

llm = ChatOpenAI(model="gpt-4o-mini")

# 专业 Agent: 研究员
@tool
def web_search(query: str) -> str:
    """搜索网络信息"""
    return f"搜索 '{query}' 的结果：这是一些相关信息..."

researcher = create_react_agent(
    llm,
    tools=[web_search],
    prompt="你是一个研究员，擅长搜索和整理信息。请用中文回答。"
)

# 专业 Agent: 写作者
writer_llm = ChatOpenAI(model="gpt-4o-mini")

def writer_node(state: State):
    response = writer_llm.invoke([
        {"role": "system", "content": "你是一个专业写作者，擅长将信息整理成清晰的文章。"},
        *state["messages"]
    ])
    return {"messages": [response]}

# Supervisor: 决定下一步
def supervisor(state: State):
    response = llm.invoke([
        {"role": "system", "content": """你是一个任务调度员。根据对话内容决定下一步：
- 需要搜索信息 → 返回 "researcher"
- 需要撰写内容 → 返回 "writer"
- 任务已完成 → 返回 "FINISH"
只返回一个词。"""},
        *state["messages"]
    ])
    next_agent = response.content.strip().lower()
    return {"next_agent": next_agent}

def route_supervisor(state: State) -> str:
    next_agent = state.get("next_agent", "FINISH")
    if "researcher" in next_agent:
        return "researcher"
    if "writer" in next_agent:
        return "writer"
    return END

# 构建图
graph_builder = StateGraph(State)

graph_builder.add_node("supervisor", supervisor)
graph_builder.add_node("researcher", researcher)
graph_builder.add_node("writer", writer_node)

graph_builder.add_edge(START, "supervisor")
graph_builder.add_conditional_edges("supervisor", route_supervisor)
graph_builder.add_edge("researcher", "supervisor")
graph_builder.add_edge("writer", "supervisor")

graph = graph_builder.compile()
```

### 模式二：协作模式

Agent 之间直接传递信息，无需中央调度：

```
[Agent A] ←→ [Agent B]
    ↕            ↕
[Agent C] ←→ [Agent D]
```

```python
def agent_a(state: State):
    """分析 Agent"""
    response = llm.invoke([
        {"role": "system", "content": "你负责分析问题，将复杂问题分解为子任务。"},
        *state["messages"]
    ])
    return {"messages": [response]}

def agent_b(state: State):
    """执行 Agent"""
    response = llm.invoke([
        {"role": "system", "content": "你负责执行具体任务，根据分析结果行动。"},
        *state["messages"]
    ])
    return {"messages": [response]}

def route_after_a(state: State) -> str:
    last_msg = state["messages"][-1].content
    if "需要执行" in last_msg:
        return "agent_b"
    return END

graph_builder = StateGraph(State)
graph_builder.add_node("agent_a", agent_a)
graph_builder.add_node("agent_b", agent_b)

graph_builder.add_edge(START, "agent_a")
graph_builder.add_conditional_edges("agent_a", route_after_a)
graph_builder.add_edge("agent_b", "agent_a")  # 执行后回到分析

graph = graph_builder.compile()
```

### 模式三：层级模式

将 Supervisor 嵌套，形成多层管理结构：

```
[总管理] → [研究组长] → [搜索员]
                     → [分析员]
         → [写作组长] → [草稿员]
                     → [编辑员]
```

```python
# 第一层：研究团队子图
research_builder = StateGraph(State)
research_builder.add_node("search_agent", search_node)
research_builder.add_node("analysis_agent", analysis_node)
research_builder.add_node("research_lead", research_lead_node)
# ... 添加边 ...
research_team = research_builder.compile()

# 第一层：写作团队子图
writing_builder = StateGraph(State)
writing_builder.add_node("draft_agent", draft_node)
writing_builder.add_node("edit_agent", edit_node)
writing_builder.add_node("writing_lead", writing_lead_node)
# ... 添加边 ...
writing_team = writing_builder.compile()

# 第二层：总管理图
main_builder = StateGraph(State)
main_builder.add_node("top_supervisor", top_supervisor_node)
main_builder.add_node("research_team", research_team)
main_builder.add_node("writing_team", writing_team)
# ... 添加边 ...
main_graph = main_builder.compile()
```

## Agent 间通信

### 通过共享状态

最常见的方式——所有 Agent 通过状态的 `messages` 字段通信：

```python
class SharedState(TypedDict):
    messages: Annotated[list, add_messages]  # 共享消息历史
    research_notes: str                       # 研究 Agent 的输出
    draft: str                                # 写作 Agent 的输出
```

### 通过独立 Channel

为不同 Agent 分配独立的状态字段，减少干扰：

```python
class MultiAgentState(TypedDict):
    user_request: str
    researcher_output: str
    writer_output: str
    final_result: str

def researcher(state: MultiAgentState):
    # 只读 user_request，只写 researcher_output
    result = do_research(state["user_request"])
    return {"researcher_output": result}

def writer(state: MultiAgentState):
    # 读 researcher_output，写 writer_output
    draft = write_article(state["researcher_output"])
    return {"writer_output": draft}
```

## 小结

| 模式 | 特点 | 适用场景 |
|------|------|----------|
| 子图 | 封装独立子流程 | 模块化复用 |
| Supervisor | 中央调度 | 任务分配明确 |
| 协作 | Agent 间直接通信 | 对等协作 |
| 层级 | 多层 Supervisor | 大型复杂系统 |
