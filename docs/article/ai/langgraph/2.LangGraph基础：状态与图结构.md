# LangGraph 基础：状态与图结构

上一篇我们快速构建了一个对话机器人。本篇将深入理解 LangGraph 的核心——**状态（State）**，以及图的执行机制。

## 状态的本质

在 LangGraph 中，状态是所有节点共享的**数据容器**。每个节点从状态中读取数据，处理后将更新写回状态。整个图的执行就是状态被逐步修改的过程。

```
状态(v0) → [节点A] → 状态(v1) → [节点B] → 状态(v2) → ...
```

### 定义状态

使用 `TypedDict` 定义状态的字段：

```python
from typing_extensions import TypedDict

class State(TypedDict):
    question: str
    context: str
    answer: str
```

每个字段就是状态中的一个 **Channel**。节点可以读取任意字段，返回需要更新的字段。

### Reducer：控制状态如何更新

默认情况下，节点返回的值会**覆盖**状态中的对应字段。但有些场景需要不同的更新策略，比如消息列表应该是追加而非覆盖。

这就是 **Reducer** 的作用——它定义了一个字段被更新时的合并策略。

```python
from typing import Annotated
from operator import add

class State(TypedDict):
    messages: Annotated[list, add_messages]  # 追加合并
    count: Annotated[int, add]               # 数值累加
    title: str                               # 默认覆盖
```

#### 常用 Reducer

| Reducer | 行为 | 适用场景 |
|---------|------|----------|
| 无（默认） | 新值覆盖旧值 | 单值字段 |
| `add_messages` | 智能追加消息列表 | 对话历史 |
| `operator.add` | 数值/列表累加 | 计数器、列表聚合 |
| 自定义函数 | 自定义合并逻辑 | 去重、条件合并等 |

#### add_messages 的智能行为

`add_messages` 不仅是简单追加，它还支持按 `id` 去重更新：

```python
from langgraph.graph.message import add_messages
from langchain_core.messages import AIMessage

existing = [AIMessage(content="你好", id="msg1")]
new = [AIMessage(content="你好！很高兴认识你", id="msg1")]

result = add_messages(existing, new)
# 结果：[AIMessage(content="你好！很高兴认识你", id="msg1")]
# msg1 被更新而非重复追加
```

### 自定义 Reducer

```python
def deduplicate(existing: list[str], new: list[str]) -> list[str]:
    """去重合并"""
    return list(set(existing + new))

class State(TypedDict):
    tags: Annotated[list[str], deduplicate]
```

## 图的生命周期

一个 StateGraph 的完整生命周期：

### 1. 定义阶段

```python
from langgraph.graph import StateGraph

graph_builder = StateGraph(State)

# 添加节点
graph_builder.add_node("retrieve", retrieve_docs)
graph_builder.add_node("generate", generate_answer)

# 添加边
graph_builder.add_edge(START, "retrieve")
graph_builder.add_edge("retrieve", "generate")
graph_builder.add_edge("generate", END)
```

### 2. 编译阶段

```python
graph = graph_builder.compile()
```

编译时会验证图的完整性：
- 所有节点必须有至少一条入边和出边
- 不能存在不可达节点
- `START` 必须有出边，`END` 必须有入边

### 3. 执行阶段

```python
result = graph.invoke({"question": "什么是 LangGraph？"})
```

执行流程：

1. 用输入初始化状态
2. 从 `START` 开始，按边的定义顺序执行节点
3. 每个节点接收当前状态，返回更新
4. 更新通过 Reducer 合并到状态
5. 到达 `END` 时返回最终状态

## 实战：构建一个 RAG 问答流程

将上述知识串联起来，构建一个「检索-生成」流程：

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv

load_dotenv()

class RAGState(TypedDict):
    question: str
    context: str
    answer: str

llm = ChatOpenAI(model="gpt-4o-mini")

def retrieve(state: RAGState):
    """模拟文档检索"""
    question = state["question"]
    # 实际场景中这里会调用向量数据库
    mock_docs = {
        "LangGraph": "LangGraph 是一个用于构建有状态 LLM 应用的框架，基于图结构。",
        "default": "未找到相关文档。"
    }
    context = next(
        (v for k, v in mock_docs.items() if k.lower() in question.lower()),
        mock_docs["default"]
    )
    return {"context": context}

def generate(state: RAGState):
    """基于检索结果生成回答"""
    prompt = f"""请基于以下上下文回答问题。

上下文：{state['context']}

问题：{state['question']}

请用简洁清晰的语言回答："""

    response = llm.invoke(prompt)
    return {"answer": response.content}

# 构建图
graph_builder = StateGraph(RAGState)
graph_builder.add_node("retrieve", retrieve)
graph_builder.add_node("generate", generate)

graph_builder.add_edge(START, "retrieve")
graph_builder.add_edge("retrieve", "generate")
graph_builder.add_edge("generate", END)

graph = graph_builder.compile()

# 执行
result = graph.invoke({"question": "什么是 LangGraph？"})
print(f"问题：{result['question']}")
print(f"上下文：{result['context']}")
print(f"回答：{result['answer']}")
```

## 状态作用域与数据流

理解数据在图中的流动方式很重要：

```python
def node_a(state: State):
    # 读取：可以访问状态中所有字段
    question = state["question"]

    # 写入：只返回需要更新的字段
    return {"context": "some context"}
    # 不需要返回 question，它不会被修改
```

**关键规则**：

1. 节点可以读取状态中的**任意字段**
2. 节点只需返回**要更新的字段**，未返回的字段保持不变
3. 返回值通过对应字段的 Reducer 合并到状态中
4. 如果返回 `None` 或空字典，状态不会变化

## 多值状态与复杂结构

状态字段可以是任意可序列化类型：

```python
from typing import Annotated
from typing_extensions import TypedDict

class ComplexState(TypedDict):
    messages: Annotated[list, add_messages]  # 消息列表
    metadata: dict                           # 嵌套字典
    scores: list[float]                      # 数值列表
    is_complete: bool                        # 布尔标志
    retry_count: int                         # 计数器
```

::: warning 注意
状态中的值需要能被序列化（用于持久化和检查点），避免存储不可序列化的对象（如数据库连接、文件句柄等）。
:::
