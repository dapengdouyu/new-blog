# LangGraph 进阶：工具调用与 Agent

Agent 是 LangGraph 最核心的应用场景。一个 Agent 能够自主决定是否调用工具、调用哪个工具，并基于工具返回的结果继续推理。本篇将教你构建一个完整的 ReAct Agent。

## ReAct 模式

ReAct（Reasoning + Acting）是最经典的 Agent 模式：

1. **推理（Reason）**：LLM 分析当前状态，决定下一步行动
2. **行动（Act）**：调用工具执行具体操作
3. **观察（Observe）**：将工具结果反馈给 LLM
4. 循环以上步骤，直到 LLM 认为可以给出最终答案

```
[agent] → 需要工具 → [tools] → [agent] → 需要工具 → [tools] → [agent] → 最终回答 → [END]
```

## 定义工具

LangGraph 使用 LangChain 的 `@tool` 装饰器定义工具：

```python
from langchain_core.tools import tool

@tool
def search(query: str) -> str:
    """搜索互联网获取最新信息。当需要查找实时信息时使用此工具。"""
    # 实际场景中调用搜索 API
    return f"搜索 '{query}' 的结果：LangGraph 是 LangChain 团队开发的框架..."

@tool
def calculator(expression: str) -> str:
    """计算数学表达式。当需要进行数学计算时使用此工具。"""
    try:
        result = eval(expression)  # 生产环境应使用安全的表达式解析器
        return str(result)
    except Exception as e:
        return f"计算错误: {e}"

tools = [search, calculator]
```

::: warning 工具的 docstring 很重要
LLM 通过 docstring 理解工具的用途和使用场景。写清楚、具体的描述能显著提高工具调用的准确性。
:::

## 将工具绑定到 LLM

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o-mini")
llm_with_tools = llm.bind_tools(tools)
```

`bind_tools` 将工具的定义（名称、描述、参数）传递给 LLM，让它知道有哪些工具可用。

## 构建 Agent 图

### 完整代码

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langgraph.prebuilt import ToolNode
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from dotenv import load_dotenv

load_dotenv()

# 1. 定义状态
class State(TypedDict):
    messages: Annotated[list, add_messages]

# 2. 定义工具
@tool
def search(query: str) -> str:
    """搜索互联网获取最新信息"""
    return f"搜索结果：LangGraph 是一个构建有状态 LLM 应用的框架。"

@tool
def calculator(expression: str) -> str:
    """计算数学表达式"""
    return str(eval(expression))

tools = [search, calculator]

# 3. 创建带工具的 LLM
llm = ChatOpenAI(model="gpt-4o-mini")
llm_with_tools = llm.bind_tools(tools)

# 4. 定义节点
def agent(state: State):
    response = llm_with_tools.invoke(state["messages"])
    return {"messages": [response]}

tool_node = ToolNode(tools)

# 5. 定义路由
def should_continue(state: State) -> str:
    last_message = state["messages"][-1]
    if hasattr(last_message, "tool_calls") and last_message.tool_calls:
        return "tools"
    return END

# 6. 构建图
graph_builder = StateGraph(State)

graph_builder.add_node("agent", agent)
graph_builder.add_node("tools", tool_node)

graph_builder.add_edge(START, "agent")
graph_builder.add_conditional_edges("agent", should_continue)
graph_builder.add_edge("tools", "agent")

graph = graph_builder.compile()
```

### 关键组件解析

#### ToolNode

`ToolNode` 是 LangGraph 预置的节点，它会：
1. 读取最后一条 AI 消息中的 `tool_calls`
2. 执行对应的工具函数
3. 将结果作为 `ToolMessage` 写入消息列表

```python
from langgraph.prebuilt import ToolNode

tool_node = ToolNode(tools)
```

#### should_continue 路由

Agent 的核心决策逻辑：

```python
def should_continue(state: State) -> str:
    last_message = state["messages"][-1]
    # 如果 AI 消息包含工具调用 → 继续执行工具
    if hasattr(last_message, "tool_calls") and last_message.tool_calls:
        return "tools"
    # 否则 → 结束
    return END
```

### 执行流程

```python
result = graph.invoke({
    "messages": [{"role": "user", "content": "123 * 456 等于多少？"}]
})

for msg in result["messages"]:
    print(f"{msg.type}: {msg.content}")
```

输出：

```
human: 123 * 456 等于多少？
ai:                              ← AI 决定调用 calculator 工具
tool: 56088                      ← 工具返回计算结果
ai: 123 × 456 = 56,088          ← AI 基于工具结果给出最终回答
```

## 使用预置的 create_react_agent

对于标准的 ReAct Agent，LangGraph 提供了快捷方式：

```python
from langgraph.prebuilt import create_react_agent
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o-mini")
graph = create_react_agent(llm, tools)

result = graph.invoke({
    "messages": [{"role": "user", "content": "搜索 LangGraph 的最新版本"}]
})
```

`create_react_agent` 自动创建了上面手动构建的完整图结构。适合标准场景快速开发。

### 添加系统提示

```python
graph = create_react_agent(
    llm,
    tools,
    prompt="你是一个专业的技术助手，请用中文回答问题。回答要准确、简洁。"
)
```

## 多工具协作示例

当 Agent 需要组合多个工具来完成任务：

```python
@tool
def get_weather(city: str) -> str:
    """获取城市天气信息"""
    weather_data = {"北京": "晴，25°C", "上海": "多云，28°C"}
    return weather_data.get(city, f"{city}：暂无数据")

@tool
def get_population(city: str) -> str:
    """获取城市人口信息"""
    pop_data = {"北京": "2189万", "上海": "2487万"}
    return pop_data.get(city, f"{city}：暂无数据")

@tool
def calculator(expression: str) -> str:
    """计算数学表达式"""
    return str(eval(expression))

tools = [get_weather, get_population, calculator]
graph = create_react_agent(ChatOpenAI(model="gpt-4o-mini"), tools)

result = graph.invoke({
    "messages": [{"role": "user", "content": "北京和上海哪个人口更多？多多少？今天天气分别怎样？"}]
})
```

Agent 会自动规划调用顺序：
1. 调用 `get_population("北京")`
2. 调用 `get_population("上海")`
3. 调用 `calculator("2487 - 2189")` 计算差值
4. 调用 `get_weather("北京")`
5. 调用 `get_weather("上海")`
6. 综合所有结果给出回答

## 错误处理

当工具执行失败时，`ToolNode` 默认会将错误信息作为 `ToolMessage` 返回给 Agent，Agent 可以选择重试或给出错误提示。

你也可以启用自动错误处理：

```python
tool_node = ToolNode(tools, handle_tool_errors=True)
```

## 小结

| 概念 | 说明 |
|------|------|
| `@tool` | 定义工具函数，docstring 作为工具描述 |
| `bind_tools()` | 将工具绑定到 LLM |
| `ToolNode` | 预置节点，自动执行工具调用 |
| `should_continue` | 路由函数，判断是否需要继续调用工具 |
| `create_react_agent` | 快捷方式，自动构建 ReAct Agent |
