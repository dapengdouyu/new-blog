# Deep Agents 高级：Deep Research 实战

Deep Research 是 Deep Agents 最具代表性的高级应用场景。一个 Deep Research Agent 能够**自主规划研究方向、并行调度多个子 Agent 检索信息、最终合成一份结构化研究报告**。本篇将手把手带你构建一个完整的 Deep Research 系统。

## 什么是 Deep Research

传统搜索只能返回单一查询的结果。Deep Research 则是一种多步骤、多来源的智能研究流程：

1. **规划**：将用户的研究问题分解为多个子问题
2. **并行检索**：为每个子问题分配独立的子 Agent，并行搜索和阅读文档
3. **合成**：将所有子 Agent 的研究结果汇总，生成一份完整的研究报告

```
                         ┌→ [子Agent: 子问题1] → 检索 + 阅读 ─┐
[用户问题] → [规划Agent] ─┼→ [子Agent: 子问题2] → 检索 + 阅读 ─┼→ [合成Agent] → 报告
                         └→ [子Agent: 子问题3] → 检索 + 阅读 ─┘
```

这种架构的优势在于：
- **覆盖面广**：多个子问题并行研究，不遗漏关键维度
- **深度足够**：每个子 Agent 可以多轮搜索、逐步深入
- **结构清晰**：最终报告基于有组织的子研究结果合成

## 第一步：定义研究工具

研究 Agent 需要两个核心工具——**网络搜索**和**文档阅读**。

```python
from langchain_core.tools import tool
from tavily import TavilyClient

tavily_client = TavilyClient()


@tool
def web_search(query: str) -> str:
    """搜索互联网获取某个主题的最新信息。传入一个清晰、具体的搜索查询。"""
    try:
        results = tavily_client.search(
            query=query,
            max_results=5,
            search_depth="advanced",
        )
        output_parts = []
        for r in results.get("results", []):
            output_parts.append(
                f"标题: {r['title']}\n"
                f"来源: {r['url']}\n"
                f"内容: {r['content']}\n"
            )
        return "\n---\n".join(output_parts) if output_parts else "未找到相关结果"
    except Exception as e:
        return f"搜索出错: {e}"


@tool
def read_document(url: str) -> str:
    """读取指定 URL 的网页内容，提取正文文本。用于深入阅读搜索结果中的文章。"""
    try:
        result = tavily_client.extract(urls=[url])
        contents = result.get("results", [])
        if contents:
            text = contents[0].get("raw_content", "")
            # 截取前 3000 字符，避免上下文溢出
            return text[:3000] if text else "页面内容为空"
        return "无法提取页面内容"
    except Exception as e:
        return f"读取文档出错: {e}"


research_tools = [web_search, read_document]
```

## 第二步：定义研究子 Agent

每个子 Agent 负责研究一个特定的子问题。我们使用 `create_deep_agent` 创建一个专注于单一子问题的研究者。

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")


def create_sub_researcher(subtopic: str):
    """为特定子问题创建一个研究子 Agent。"""
    system_prompt = f"""你是一个专业研究员，专注于以下研究子问题：

{subtopic}

你的工作流程：
1. 使用 web_search 工具搜索相关信息（至少搜索 2-3 次，使用不同关键词）
2. 对搜索结果中最相关的文章，使用 read_document 深入阅读
3. 整理你的发现，输出一份结构化的子研究报告

子研究报告格式：
## 研究发现：{subtopic}
### 关键信息
- 要点 1
- 要点 2
### 数据与来源
- 引用具体数据和来源 URL
### 小结
一段话总结本子问题的核心发现。
"""

    agent = create_deep_agent(
        model=llm,
        tools=research_tools,
        system_prompt=system_prompt,
    )
    return agent
```

## 第三步：构建主研究 Agent

主 Agent 负责三件事：**规划子问题**、**调度子 Agent**、**合成最终报告**。

### 规划器：分解研究问题

```python
from langchain_core.output_parsers import JsonOutputParser


async def plan_research(topic: str) -> list[str]:
    """将用户的研究主题分解为 3-5 个子问题。"""
    planner_prompt = f"""你是一个研究规划专家。请将以下研究主题分解为 3-5 个具体的子问题，
每个子问题应该可以独立研究，合在一起能全面覆盖主题。

研究主题：{topic}

以 JSON 数组格式返回子问题列表，例如：
["子问题1", "子问题2", "子问题3"]

只返回 JSON，不要其他内容。"""

    response = await llm.ainvoke(planner_prompt)
    parser = JsonOutputParser()
    subtopics = parser.parse(response.content)
    return subtopics
```

### 并行执行子研究

```python
import asyncio


async def run_parallel_research(subtopics: list[str]) -> list[str]:
    """并行运行多个子 Agent，各自研究一个子问题。"""

    async def research_one(subtopic: str) -> str:
        agent = create_sub_researcher(subtopic)
        result = await agent.ainvoke(
            {"messages": [{"role": "user", "content": f"请深入研究以下问题：{subtopic}"}]}
        )
        # 提取最终回复
        final_message = result["messages"][-1]
        return final_message.content

    # 并行启动所有子 Agent
    tasks = [research_one(sub) for sub in subtopics]
    results = await asyncio.gather(*tasks)
    return list(results)
```

### 合成器：生成最终报告

```python
async def synthesize_report(topic: str, sub_results: list[str]) -> str:
    """将所有子研究结果合成为一份完整的研究报告。"""
    combined = "\n\n===\n\n".join(sub_results)

    synthesis_prompt = f"""你是一个高级研究分析师。以下是多个研究员对主题「{topic}」各子问题的独立研究结果。

请基于这些研究结果，合成一份完整、结构化的研究报告。

要求：
1. 报告需要有清晰的标题和章节结构
2. 综合多个子研究的发现，去除重复，建立联系
3. 关键结论需要引用数据或来源
4. 最后给出总结和建议

各子研究结果：
{combined}

请输出完整的研究报告（Markdown 格式）。"""

    response = await llm.ainvoke(synthesis_prompt)
    return response.content
```

## 第四步：组装完整的 Deep Research 流程

将规划、并行研究、合成三个阶段串联起来：

```python
async def deep_research(topic: str) -> str:
    """运行完整的 Deep Research 流程。"""
    print(f"🔬 开始研究主题：{topic}\n")

    # 阶段 1：规划
    print("📋 阶段 1/3：规划研究子问题...")
    subtopics = await plan_research(topic)
    print(f"   分解为 {len(subtopics)} 个子问题：")
    for i, sub in enumerate(subtopics, 1):
        print(f"   {i}. {sub}")
    print()

    # 阶段 2：并行研究
    print("🔍 阶段 2/3：并行执行子研究...")
    sub_results = await run_parallel_research(subtopics)
    print(f"   完成 {len(sub_results)} 个子研究\n")

    # 阶段 3：合成报告
    print("📝 阶段 3/3：合成最终报告...")
    report = await synthesize_report(topic, sub_results)
    print("   报告生成完毕\n")

    return report


# 运行示例
if __name__ == "__main__":
    topic = "2025 年主流 Python Web 框架对比分析（Django, FastAPI, Flask, Litestar）"
    report = asyncio.run(deep_research(topic))
    print("=" * 60)
    print(report)
```

## 使用流式输出改进体验

在实际应用中，研究过程可能需要几分钟。通过流式输出让用户实时看到进度：

```python
async def deep_research_streaming(topic: str):
    """带流式输出的 Deep Research。"""
    print(f"开始研究：{topic}\n")

    # 规划阶段
    subtopics = await plan_research(topic)
    for i, sub in enumerate(subtopics, 1):
        print(f"子问题 {i}: {sub}")

    # 并行研究阶段，使用 stream 实时输出
    async def research_and_stream(subtopic: str, index: int):
        agent = create_sub_researcher(subtopic)
        print(f"\n--- 子研究 {index} 开始：{subtopic} ---")
        async for event in agent.astream(
            {"messages": [{"role": "user", "content": f"请深入研究：{subtopic}"}]},
            stream_mode="updates",
        ):
            for node_name, update in event.items():
                if node_name == "agent":
                    messages = update.get("messages", [])
                    for msg in messages:
                        if hasattr(msg, "content") and msg.content:
                            print(f"  [{index}] {msg.content[:100]}...")

    tasks = [research_and_stream(sub, i) for i, sub in enumerate(subtopics, 1)]
    await asyncio.gather(*tasks)

    print("\n正在合成最终报告...")
```

## 进阶：添加质量检查环节

在子研究完成后、合成之前，增加一个质量检查步骤，确保研究结果的可靠性：

```python
async def quality_check(subtopic: str, result: str) -> str:
    """检查子研究结果的质量，必要时补充研究。"""
    check_prompt = f"""请评估以下研究结果的质量：

研究问题：{subtopic}
研究结果：
{result}

评估标准：
1. 是否回答了研究问题？
2. 是否有具体数据支撑？
3. 是否引用了可靠来源？

如果质量合格，返回 "PASS"。
如果需要补充，返回 "NEED_MORE: <具体需要补充什么>"。"""

    response = await llm.ainvoke(check_prompt)
    return response.content


async def research_with_quality(subtopics: list[str]) -> list[str]:
    """带质量检查的并行研究。"""

    async def research_one_with_check(subtopic: str) -> str:
        agent = create_sub_researcher(subtopic)
        result = await agent.ainvoke(
            {"messages": [{"role": "user", "content": f"请深入研究：{subtopic}"}]}
        )
        content = result["messages"][-1].content

        # 质量检查
        check = await quality_check(subtopic, content)
        if check.startswith("NEED_MORE"):
            supplement_query = check.replace("NEED_MORE:", "").strip()
            result = await agent.ainvoke(
                {"messages": [
                    {"role": "user", "content": f"请补充研究以下方面：{supplement_query}"}
                ]}
            )
            content += "\n\n### 补充研究\n" + result["messages"][-1].content

        return content

    tasks = [research_one_with_check(sub) for sub in subtopics]
    return list(await asyncio.gather(*tasks))
```

## 提升研究质量的技巧

| 技巧 | 说明 | 实现方式 |
|------|------|----------|
| 多轮搜索 | 子 Agent 使用不同关键词多次搜索 | 在系统提示词中要求至少搜索 2-3 次 |
| 来源验证 | 对搜索结果交叉验证 | 增加 quality_check 环节 |
| 深入阅读 | 不仅看搜索摘要，还要阅读原文 | 使用 read_document 工具 |
| 结构化输出 | 子研究结果格式统一 | 在系统提示词中定义输出格式 |
| 迭代补充 | 质量不达标时自动补充 | quality_check + 重新研究 |
| 模型选择 | 规划和合成用强模型，搜索用快模型 | 不同阶段使用不同 ChatModel |

::: warning 成本控制
Deep Research 会产生大量 LLM 调用。建议在开发阶段使用 gpt-4o-mini 降低成本，确认流程正确后再切换到 gpt-4o。同时设置合理的 recursion_limit 防止子 Agent 无限循环。
:::
