# Deep Agents 入门：认识深度 Agent

本篇将带你理解什么是"深度 Agent"、它与传统 Agent 的本质区别，以及 LangChain 官方 Deep Agents 框架的核心架构。最后你将完成环境搭建并运行第一个 Deep Agent。

## 什么是 Agent？

在 LLM 应用中，Agent 是一种让模型**自主决策、调用工具、循环推理**的模式。最简单的 Agent 就是一个 while 循环：

```
while True:
    response = llm.invoke(messages)
    if response.has_tool_calls:
        result = execute_tools(response.tool_calls)
        messages.append(result)
    else:
        break  # 模型认为任务完成
```

这种基础 Agent 能处理简单任务——比如查天气、做计算。但面对复杂的、需要多步骤规划的任务时，它就力不从心了。

## 浅层 Agent vs 深度 Agent

**浅层 Agent（Shallow Agent）** 就是上面描述的简单循环：LLM 接收输入，决定调用哪个工具，拿到结果后直接回答。整个过程通常在 2-5 轮内结束。

**深度 Agent（Deep Agent）** 则具备更强的能力：它能规划多步任务、将大问题拆解成子任务、委派给子代理执行、通过文件系统管理中间产物，并在数十甚至上百轮交互中持续工作。

```
浅层 Agent：
用户提问 → LLM → 调用 1-2 个工具 → 返回答案

深度 Agent：
用户提问 → 制定计划 → 拆解子任务 → 委派子代理 → 读写文件 → 反思调整 → ... → 交付成果
```

### 现实中的深度 Agent

这些产品本质上都是深度 Agent：

- **Deep Research**（OpenAI / Google）：接受一个研究主题，自主搜索数十篇文献，整理成完整报告
- **Claude Code**（Anthropic）：接受一个编程任务，自主阅读代码库、规划方案、编写代码、运行测试
- **Manus**：接受一个复杂指令，自主浏览网页、操作文件、执行代码，完成端到端的工作流

它们的共同特点是：**不是"一问一答"，而是"接受任务，自主完成"。**

## 深度 Agent 的四大特征

是什么让一个 Agent 从"浅层"变成"深度"的？归纳下来有四个关键特征：

### 1. 详细的系统提示词（System Prompt）

浅层 Agent 的系统提示词通常只有几句话。深度 Agent 则拥有**长达数千 Token 的系统提示词**，其中包含：

- 明确的角色定义和行为准则
- 每个工具的详细使用指南和最佳实践
- 典型工作流的示例
- 错误处理和边界情况的指导

好的系统提示词就像一本操作手册，让模型"知道该怎么高效地工作"。

### 2. 规划工具（Planning Tool）

深度 Agent 拥有一个专门用于规划的工具。它可以：

- 将大任务拆解为具体的 Todo 列表
- 在执行过程中追踪进度
- 根据执行结果动态调整计划

这就像给 Agent 一块白板——它可以先规划，再执行。

### 3. 子代理（Sub-agents）

面对复杂任务时，深度 Agent 可以**派生子代理**来处理特定子任务。每个子代理拥有独立的上下文窗口，专注于一个小任务，完成后将结果汇报给主代理。

这解决了两个问题：

- **上下文窗口限制**：避免单个对话中塞入过多信息
- **关注点分离**：每个子代理只需关注自己的子任务

### 4. 文件系统访问（File System Access）

深度 Agent 能读写文件、浏览目录、搜索内容。这意味着：

- 中间结果可以写入文件，释放上下文窗口的空间
- 可以处理超长文本（逐文件阅读而非全部塞入 Prompt）
- 最终成果以文件形式交付

```
四大特征总览：

+--------------------------------------------------+
|               深度 Agent (Deep Agent)              |
|                                                    |
|  +------------+  +------------+  +--------------+  |
|  | 系统提示词  |  |  规划工具   |  |   子代理     |  |
|  | (详细指导)  |  | (Todo 列表) |  | (任务委派)   |  |
|  +------------+  +------------+  +--------------+  |
|                                                    |
|  +----------------------------------------------+  |
|  |             文件系统访问                        |  |
|  |  read_file / write_file / edit_file / grep    |  |
|  +----------------------------------------------+  |
+--------------------------------------------------+
```

## Deep Agents 框架概览

**Deep Agents** 是 LangChain 官方推出的开源框架，它将上述四大特征封装为一个**开箱即用的 Agent 解决方案**。

核心特点：

- **一行代码创建 Agent**：`create_deep_agent()` 返回一个编译好的 LangGraph 图
- **内置全套工具**：规划（`write_todos`）、文件操作（`read_file`、`write_file`、`edit_file`）、搜索（`glob`、`grep`）、Shell（`execute`）、子代理（`task`）
- **Smart Defaults**：系统提示词预置了每个工具的使用最佳实践
- **自动上下文管理**：对话过长时自动摘要，大输出自动存文件
- **模型无关**：支持 OpenAI、Anthropic、Google 等任意支持 Tool Calling 的模型

### 架构图

Deep Agents 的核心是一个 **ReAct 循环**，在 LangGraph 中表现为一个带有条件边的图：

```
                  +------------------+
                  |     START        |
                  +--------+---------+
                           |
                           v
                  +--------+---------+
          +------>|    Agent (LLM)   |<------+
          |       +--------+---------+       |
          |                |                 |
          |         有工具调用？               |
          |           /       \              |
          |         Yes        No            |
          |          |          |             |
          |          v          v             |
          |  +-------+--+  +---+------+      |
          |  |  Tools    |  |   END    |      |
          |  +-------+---+  +----------+      |
          |          |                        |
          +----------+------------------------+
             工具结果返回给 Agent
```

Agent 节点调用 LLM，如果 LLM 返回了工具调用请求，就执行对应工具并将结果反馈给 LLM；如果 LLM 直接返回文本，则任务结束。

## 环境安装

### 1. 创建项目

```bash
mkdir deepagents-tutorial && cd deepagents-tutorial
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
```

### 2. 安装依赖

```bash
pip install deepagents langchain-openai python-dotenv
```

::: tip 包说明
- `deepagents`：Deep Agents 核心框架，自带 LangGraph 依赖
- `langchain-openai`：OpenAI 模型接入（也可替换为 `langchain-anthropic` 等）
- `python-dotenv`：环境变量管理
:::

### 3. 配置 API Key

创建 `.env` 文件：

```env
OPENAI_API_KEY=sk-your-api-key-here
OPENAI_BASE_URL=https://api.openai.com/v1  # 可替换为代理地址
```

## 第一个 Deep Agent

只需几行代码就能创建并运行一个深度 Agent：

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv

load_dotenv()

# 创建模型
llm = ChatOpenAI(model="gpt-4o")

# 创建深度 Agent（返回一个编译好的 LangGraph 图）
agent = create_deep_agent(llm)

# 运行 Agent
result = agent.invoke({
    "messages": [
        {"role": "user", "content": "请在当前目录创建一个 hello.txt 文件，内容为 'Hello, Deep Agent!'"}
    ]
})

# 打印最终回复
print(result["messages"][-1].content)
```

运行后，Agent 会自主调用 `write_file` 工具创建文件，然后告诉你任务已完成。这就是深度 Agent 的威力——你只需描述目标，Agent 自己规划并执行。

### 运行效果

```bash
python main.py
```

你会看到类似这样的输出：

```
我已经在当前目录创建了 hello.txt 文件，内容为 "Hello, Deep Agent!"。
```

同时当前目录下会出现一个 `hello.txt` 文件。

## 与手动构建 Agent 的对比

如果不使用 Deep Agents，你需要手动完成这些工作：

| 功能 | 手动构建 | Deep Agents |
|------|----------|-------------|
| 工具定义 | 自己编写每个工具函数 | 内置全套工具 |
| 系统提示词 | 自己编写和调优 | 预置最佳实践 |
| 规划能力 | 需要自己实现 Todo 系统 | 内置 `write_todos` 工具 |
| 子代理 | 需要自己实现任务委派 | 内置 `task` 工具 |
| 上下文管理 | 需要自己处理长对话 | 自动摘要与大输出存文件 |
| 图结构 | 需要用 LangGraph 手动搭建 | `create_deep_agent()` 一行搞定 |

## 核心概念回顾

| 概念 | 说明 |
|------|------|
| 浅层 Agent | 简单的 LLM + 工具循环，适合短任务 |
| 深度 Agent | 具备规划、委派、文件管理能力的 Agent，适合复杂任务 |
| 系统提示词 | 详细的操作手册，指导 Agent 高效工作 |
| 规划工具 | `write_todos`，将大任务拆解为可追踪的 Todo 列表 |
| 子代理 | `task` 工具，将子任务委派给独立 Agent 执行 |
| 文件系统 | `read_file` / `write_file` / `edit_file` / `glob` / `grep` |
| `create_deep_agent()` | 核心 API，一行代码创建完整的深度 Agent |
