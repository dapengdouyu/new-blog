# Deep Agents 高级：生产部署与最佳实践

Deep Agents 基于 LangGraph 构建，天然具备生产级特性。`create_deep_agent()` 返回的就是一个编译好的 LangGraph 图，可以直接使用 LangGraph 的全部能力——流式输出、持久化检查点、Studio 可视化等。本篇将介绍如何将 Deep Agents 部署到生产环境，并分享关键的最佳实践。

## LangGraph 原生集成

### create_deep_agent 的本质

`create_deep_agent()` 返回的是一个 `CompiledGraph` 对象，与手动构建的 LangGraph 图完全兼容：

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")

agent = create_deep_agent(
    model=llm,
    tools=[...],
    system_prompt="你是一个专业助手。",
)

# agent 就是一个 CompiledGraph
print(type(agent))
# <class 'langgraph.graph.state.CompiledStateGraph'>
```

这意味着你可以直接使用 `invoke()`、`stream()`、`astream()`、`astream_events()` 等所有 LangGraph API。

### 在 LangGraph Studio 中使用

由于 Deep Agents 就是标准的 LangGraph 图，你可以直接在 LangGraph Studio 中可视化和调试：

```python
# agent.py
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool

@tool
def search(query: str) -> str:
    """搜索互联网信息"""
    return f"搜索结果：{query}"

llm = ChatOpenAI(model="gpt-4o")

graph = create_deep_agent(
    model=llm,
    tools=[search],
    system_prompt="你是一个有帮助的助手。",
)
```

配置 `langgraph.json`：

```json
{
  "dependencies": ["."],
  "graphs": {
    "deep_agent": "./agent.py:graph"
  },
  "env": ".env"
}
```

```bash
pip install langgraph-cli
langgraph dev
```

启动后即可在 Studio 中看到 Agent 的图结构、实时追踪每次执行。

## 流式输出

### 基础流式：stream()

生产环境中，用户不会等 Agent 跑完所有步骤才看到结果。使用 `stream()` 逐步输出：

```python
config = {"configurable": {"thread_id": "user-123"}}

for event in agent.stream(
    {"messages": [{"role": "user", "content": "请帮我分析这个问题"}]},
    config,
    stream_mode="updates",
):
    for node_name, update in event.items():
        print(f"[{node_name}]")
        messages = update.get("messages", [])
        for msg in messages:
            if hasattr(msg, "content") and msg.content:
                print(f"  {msg.content[:200]}")
```

### Token 级流式：astream_events()

最细粒度的流式，逐 token 输出 LLM 的生成结果：

```python
async for event in agent.astream_events(
    {"messages": [{"role": "user", "content": "写一篇 Python 入门指南"}]},
    config,
    version="v2",
):
    if event["event"] == "on_chat_model_stream":
        token = event["data"]["chunk"].content
        if token:
            print(token, end="", flush=True)
```

### 结合 FastAPI 提供 SSE 接口

```python
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import json

app = FastAPI()


@app.post("/agent/chat")
async def chat(request: dict):
    thread_id = request.get("thread_id", "default")
    message = request["message"]

    async def event_stream():
        async for event in agent.astream_events(
            {"messages": [{"role": "user", "content": message}]},
            {"configurable": {"thread_id": thread_id}},
            version="v2",
        ):
            if event["event"] == "on_chat_model_stream":
                token = event["data"]["chunk"].content
                if token:
                    yield f"data: {json.dumps({'token': token}, ensure_ascii=False)}\n\n"
            elif event["event"] == "on_tool_start":
                tool_name = event.get("name", "unknown")
                yield f"data: {json.dumps({'tool': tool_name, 'status': 'start'})}\n\n"
            elif event["event"] == "on_tool_end":
                tool_name = event.get("name", "unknown")
                yield f"data: {json.dumps({'tool': tool_name, 'status': 'done'})}\n\n"
        yield "data: [DONE]\n\n"

    return StreamingResponse(event_stream(), media_type="text/event-stream")
```

## 检查点与持久化

检查点允许 Agent 在中断后恢复执行，是生产环境中的核心需求。

### MemorySaver（开发用）

最简单的内存检查点，适合开发和测试：

```python
from langgraph.checkpoint.memory import MemorySaver

memory = MemorySaver()

agent = create_deep_agent(
    model=llm,
    tools=tools,
    system_prompt="你是一个助手。",
    checkpointer=memory,
)

config = {"configurable": {"thread_id": "session-001"}}

# 第一轮对话
result1 = agent.invoke(
    {"messages": [{"role": "user", "content": "我叫张三"}]},
    config,
)

# 第二轮对话 —— Agent 记得上一轮内容
result2 = agent.invoke(
    {"messages": [{"role": "user", "content": "我叫什么名字？"}]},
    config,
)
print(result2["messages"][-1].content)
# "你叫张三。"
```

### SqliteSaver（单机生产）

适合单机部署场景，检查点持久化到 SQLite 文件：

```python
from langgraph.checkpoint.sqlite.aio import AsyncSqliteSaver


async def create_agent_with_sqlite():
    async with AsyncSqliteSaver.from_conn_string("checkpoints.db") as saver:
        agent = create_deep_agent(
            model=llm,
            tools=tools,
            system_prompt="你是一个助手。",
            checkpointer=saver,
        )
        # 即使服务重启，之前的对话状态也不会丢失
        result = await agent.ainvoke(
            {"messages": [{"role": "user", "content": "继续我们之前的对话"}]},
            {"configurable": {"thread_id": "session-001"}},
        )
        return result
```

### PostgresSaver（分布式生产）

多实例部署时使用 PostgreSQL，多个服务实例共享状态：

```python
from langgraph.checkpoint.postgres.aio import AsyncPostgresSaver

DB_URI = "postgresql://user:password@localhost:5432/agent_db"


async def create_agent_with_postgres():
    async with AsyncPostgresSaver.from_conn_string(DB_URI) as saver:
        await saver.setup()  # 首次运行时创建表结构

        agent = create_deep_agent(
            model=llm,
            tools=tools,
            system_prompt="你是一个助手。",
            checkpointer=saver,
        )
        return agent
```

| 检查点类型 | 适用场景 | 特点 |
|-----------|---------|------|
| MemorySaver | 开发、测试 | 内存存储，重启丢失 |
| SqliteSaver | 单机部署 | 文件存储，轻量持久 |
| PostgresSaver | 分布式部署 | 多实例共享，生产推荐 |

## Deep Agents CLI

Deep Agents 提供了一个开箱即用的 **CLI 编码代理**，可以在终端中直接使用，无需编写任何代码。

### 安装

```bash
curl -sSL https://raw.githubusercontent.com/langchain-ai/deepagents/main/install.sh | bash
```

### 交互模式

安装后直接运行进入交互式 TUI（终端用户界面）：

```bash
deepagents
```

在交互模式中，你可以：
- 让 Agent 阅读和修改项目文件
- 执行 Shell 命令
- 搜索网络信息
- 进行多轮对话，逐步完善任务

### Headless 模式（CI/CD）

Headless 模式适合在 CI/CD 管道中自动化执行任务：

```bash
# 代码审查
deepagents --headless "请审查 src/ 目录下的代码，列出潜在问题"

# 自动生成文档
deepagents --headless "为 src/utils.py 中的所有函数生成 docstring"

# 自动修复 lint 错误
deepagents --headless "运行 ruff check，修复所有可自动修复的问题"
```

在 GitHub Actions 中使用：

```yaml
name: AI Code Review
on: [pull_request]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install Deep Agents CLI
        run: curl -sSL https://raw.githubusercontent.com/langchain-ai/deepagents/main/install.sh | bash
      - name: Run code review
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
        run: deepagents --headless "审查本次 PR 的改动，输出发现的问题"
```

### Human-in-the-Loop

CLI 支持在关键操作前请求人工确认：

```bash

```

## LangSmith 集成

LangSmith 为 Deep Agents 提供完整的可观测性——追踪每一次 LLM 调用、工具调用、Agent 决策。

### 基本配置

```python
import os

os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_API_KEY"] = "your-langsmith-api-key"
os.environ["LANGCHAIN_PROJECT"] = "deep-agents-prod"
```

配置后，所有通过 `agent.invoke()` 或 `agent.stream()` 的调用都会自动上报到 LangSmith。

### 在 LangSmith 中你可以看到

- **完整调用链**：从用户输入 -> Agent 推理 -> 工具调用 -> 最终回复的完整链路
- **Token 用量**：每次 LLM 调用的 Token 消耗和成本
- **延迟分析**：各步骤的耗时，定位性能瓶颈
- **错误追踪**：工具调用失败、LLM 超时等异常的详细信息
- **对话历史**：按 thread_id 查看完整对话记录

### 自定义追踪标签

```python
config = {
    "configurable": {"thread_id": "user-123"},
    "metadata": {
        "user_id": "user-123",
        "environment": "production",
        "feature": "deep_research",
    },
    "tags": ["production", "research"],
}

result = agent.invoke(
    {"messages": [{"role": "user", "content": "分析市场趋势"}]},
    config,
)
```

添加标签和元数据后，可以在 LangSmith 中按标签筛选和分析追踪记录。

## 错误处理与重试

### 工具级错误处理

在工具内部捕获异常，返回友好的错误信息而非让整个 Agent 崩溃：

```python
from langchain_core.tools import tool


@tool
def query_database(sql: str) -> str:
    """执行 SQL 查询获取数据"""
    try:
        result = db.execute(sql)
        return str(result)
    except ConnectionError:
        return "数据库连接失败，请稍后重试"
    except Exception as e:
        return f"查询出错：{e}。请检查 SQL 语法。"
```

### Agent 级重试与超时

```python
from langchain_core.runnables import RunnableConfig


async def safe_agent_call(
    agent,
    message: str,
    thread_id: str,
    max_retries: int = 3,
    timeout: float = 120.0,
):
    """带重试和超时的 Agent 调用。"""
    config = RunnableConfig(
        configurable={"thread_id": thread_id},
        recursion_limit=50,
    )

    for attempt in range(max_retries):
        try:
            result = await asyncio.wait_for(
                agent.ainvoke(
                    {"messages": [{"role": "user", "content": message}]},
                    config,
                ),
                timeout=timeout,
            )
            return result
        except asyncio.TimeoutError:
            print(f"第 {attempt + 1} 次调用超时，重试中...")
        except Exception as e:
            print(f"第 {attempt + 1} 次调用出错：{e}")
            if attempt == max_retries - 1:
                raise

    raise RuntimeError(f"Agent 调用在 {max_retries} 次重试后仍然失败")
```

### 递归限制保护

防止 Agent 陷入无限循环：

```python
from langgraph.errors import GraphRecursionError

try:
    result = agent.invoke(
        {"messages": [{"role": "user", "content": "请完成任务"}]},
        {"configurable": {"thread_id": "t1"}, "recursion_limit": 30},
    )
except GraphRecursionError:
    print("Agent 执行步数超过限制，可能陷入循环")
    # 从检查点恢复最近的有效状态
    state = agent.get_state({"configurable": {"thread_id": "t1"}})
    print(f"最后状态包含 {len(state.values.get('messages', []))} 条消息")
```

## 安全实践

### Shell 执行沙箱

Deep Agents 内置的 Shell 工具需要严格控制：

```python
from deepagents import create_deep_agent

agent = create_deep_agent(
    model=llm,
    tools=tools,
    system_prompt="你是一个助手。",
    # 限制 Shell 工具的能力
    shell_config={
        "allowed_commands": ["ls", "cat", "grep", "find", "python"],
        "blocked_commands": ["rm -rf", "sudo", "curl | bash"],
        "working_directory": "/app/workspace",
        "timeout": 30,
    },
)
```

### 工具级权限边界

为不同用户角色配置不同的工具集：

```python
def create_agent_for_role(role: str):
    """根据用户角色创建权限不同的 Agent。"""
    base_tools = [search_tool, calculator_tool]

    if role == "admin":
        tools = base_tools + [shell_tool, file_write_tool, db_tool]
    elif role == "developer":
        tools = base_tools + [file_read_tool, file_write_tool]
    elif role == "viewer":
        tools = base_tools + [file_read_tool]
    else:
        tools = base_tools

    return create_deep_agent(
        model=llm,
        tools=tools,
        system_prompt=f"你是一个助手，当前用户角色为 {role}。",
    )
```

### 敏感信息过滤

```python
import re


def sanitize_output(content: str) -> str:
    """过滤 Agent 输出中的敏感信息。"""
    patterns = [
        (r"sk-[a-zA-Z0-9]{20,}", "[API_KEY_REDACTED]"),
        (r"\b\d{4}[\s-]?\d{4}[\s-]?\d{4}[\s-]?\d{4}\b", "[CARD_REDACTED]"),
        (r"(?i)password\s*[:=]\s*\S+", "password: [REDACTED]"),
    ]
    for pattern, replacement in patterns:
        content = re.sub(pattern, replacement, content)
    return content
```

## 性能优化

### 模型选择策略

不是所有步骤都需要最强的模型。合理搭配可以大幅降低成本和延迟：

```python
from langchain_openai import ChatOpenAI

# 强模型：用于复杂推理和最终合成
strong_llm = ChatOpenAI(model="gpt-4o")

# 快速模型：用于简单分类和工具调用
fast_llm = ChatOpenAI(model="gpt-4o-mini")

# 主 Agent 使用强模型
main_agent = create_deep_agent(
    model=strong_llm,
    tools=tools,
    system_prompt="你是主 Agent，负责复杂推理。",
)

# 子 Agent 使用快速模型
sub_agent = create_deep_agent(
    model=fast_llm,
    tools=search_tools,
    system_prompt="你是搜索子 Agent。",
)
```

### 上下文窗口管理

Deep Agents 内置了上下文管理能力，但你也可以手动控制：

```python
from langchain_core.messages import trim_messages


def create_managed_agent():
    """创建带上下文管理的 Agent。"""
    agent = create_deep_agent(
        model=llm,
        tools=tools,
        system_prompt="你是一个助手。",
        # Deep Agents 内置的上下文管理配置
        context_config={
            "max_messages": 50,           # 最大消息数
            "auto_summarize": True,        # 自动摘要旧消息
            "large_output_to_file": True,  # 大输出自动存文件
        },
    )
    return agent
```

### 并发控制

多用户场景下控制并发 Agent 数量：

```python
import asyncio

# 限制同时运行的 Agent 数量
semaphore = asyncio.Semaphore(10)


async def handle_request(user_id: str, message: str):
    async with semaphore:
        result = await agent.ainvoke(
            {"messages": [{"role": "user", "content": message}]},
            {"configurable": {"thread_id": user_id}},
        )
        return result
```

## 生产部署清单

| 项目 | 要点 | 优先级 |
|------|------|--------|
| 检查点持久化 | 使用 PostgresSaver，确保会话可恢复 | 必须 |
| 流式输出 | 使用 SSE 或 WebSocket 实时返回 | 必须 |
| 错误处理 | 工具级捕获 + Agent 级重试 + 递归限制 | 必须 |
| 可观测性 | 接入 LangSmith，配置标签和元数据 | 必须 |
| 安全 | Shell 沙箱 + 工具权限 + 敏感信息过滤 | 必须 |
| 性能 | 模型分级 + 上下文管理 + 并发控制 | 推荐 |
| CI/CD | CLI headless 模式集成到流水线 | 推荐 |
| 监控告警 | LangSmith 异常检测 + 自定义日志 | 推荐 |
