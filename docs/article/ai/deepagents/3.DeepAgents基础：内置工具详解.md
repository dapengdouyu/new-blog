# Deep Agents 基础：内置工具详解

Deep Agents 之所以能够"开箱即用"，关键在于它预置了一套精心设计的内置工具。这些工具覆盖了任务规划、文件操作、命令执行和任务委派四大类场景，让 Agent 无需额外配置就能处理复杂的实际任务。本篇将逐一讲解每个内置工具的用途、参数和使用技巧。

## 内置工具总览

Deep Agents 的内置工具分为四大类：

```
Deep Agents 内置工具体系
├── 规划工具
│   └── write_todos        # 任务分解与进度追踪
├── 文件系统工具
│   ├── read_file          # 读取文件内容
│   ├── write_file         # 写入/创建文件
│   ├── edit_file          # 编辑已有文件
│   ├── ls                 # 列出目录内容
│   ├── glob               # 模式匹配搜索文件
│   └── grep               # 搜索文件内容
├── Shell 工具
│   └── execute            # 执行 Shell 命令
└── 子代理工具
    └── task               # 委派任务给子 Agent
```

## 规划工具：write_todos

### 为什么需要规划？

当面对复杂任务时，人类会先列出计划再逐步执行。`write_todos` 让 Agent 也拥有这种能力——先拆解任务，再按计划逐步推进。

### 工作原理

`write_todos` 本质上是一个 **"no-op" 上下文工程策略**。它并不会执行任何外部操作，而是将 Agent 生成的任务列表写入对话上下文中。这样做的效果是：

1. **强制 Agent 思考**：在动手之前先规划步骤
2. **提供进度追踪**：Agent 可以标记已完成的任务
3. **减少遗漏**：所有子任务被显式列出，不会被遗忘

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")
agent = create_deep_agent(llm)

# Agent 收到复杂任务后，会自动使用 write_todos 进行规划
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "分析 ./data 目录下所有 CSV 文件的数据质量，生成一份汇总报告"
    }]
})
```

Agent 会自动调用 `write_todos` 创建类似这样的计划：

```
Todo List:
- [ ] 使用 ls 和 glob 查找 ./data 下所有 CSV 文件
- [ ] 逐个读取 CSV 文件，检查数据完整性
- [ ] 统计缺失值、重复行、数据类型异常
- [ ] 生成汇总报告并写入文件
```

::: tip write_todos 的上下文工程思想
`write_todos` 是一种"无副作用"的工具——它不会修改文件系统或执行命令，只是把计划写入对话上下文。这种设计看似简单，实则巧妙：它利用了 LLM 的注意力机制，让模型在后续每一步都能"看到"完整计划，从而保持任务的连贯性和完整性。
:::

## 文件系统工具

文件系统工具让 Agent 能够自主管理工作空间中的文件。这是 Agent 完成实际工作的基础能力。

### read_file — 读取文件

读取指定路径的文件内容，支持文本文件。

```python
# Agent 在执行过程中会这样调用 read_file：
# read_file(path="./config.yaml")
# 返回文件的完整文本内容

# 实际使用中，你只需要给 Agent 下达任务
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "读取 ./config.yaml 的内容，告诉我数据库连接配置"
    }]
})
```

### write_file — 写入文件

创建新文件或完全覆盖已有文件。适用于生成新文件的场景。

```python
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "创建一个 Python 脚本 hello.py，内容是打印 Hello World"
    }]
})

# Agent 内部调用：
# write_file(path="./hello.py", content="print('Hello World')\n")
```

### edit_file — 编辑文件

对已有文件进行局部修改，而非完全覆盖。这比 `write_file` 更精细，适合修改代码或配置中的某个部分。

```python
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "把 config.yaml 中的 port: 3000 改为 port: 8080"
    }]
})

# Agent 内部调用 edit_file，只修改目标行，保留文件其余内容
```

::: warning write_file 与 edit_file 的区别
- `write_file`：**完全覆盖**文件内容，适合创建新文件
- `edit_file`：**局部修改**文件内容，适合修改已有文件

系统提示词会教导 Agent 在修改已有文件时优先使用 `edit_file`，避免误覆盖。
:::

### ls — 列出目录

列出指定目录的内容，帮助 Agent 了解工作空间的文件结构。

```python
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "看看当前目录下都有什么文件和文件夹"
    }]
})

# Agent 内部调用：
# ls(path="./")
# 返回目录列表
```

### glob — 模式匹配搜索

根据通配符模式查找文件，支持递归搜索。

```python
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "找出项目中所有的 Python 测试文件"
    }]
})

# Agent 内部调用：
# glob(pattern="**/test_*.py")
# 返回所有匹配的文件路径
```

### grep — 内容搜索

在文件内容中搜索指定文本或正则表达式，类似命令行的 `grep`。

```python
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "在 src 目录下找出所有包含 TODO 注释的文件"
    }]
})

# Agent 内部调用：
# grep(pattern="TODO", path="./src")
# 返回匹配的文件和行号
```

### 文件系统工具协作示例

Agent 通常会组合使用多个文件系统工具来完成任务：

```
用户任务：整理项目中所有的配置文件

Agent 执行流程：
[ls] 查看目录结构
  → [glob] 搜索 **/*.{yaml,yml,json,toml,ini}
    → [read_file] 逐个读取配置文件
      → [write_file] 生成汇总文档
```

## Shell 工具：execute

### 基本用法

`execute` 允许 Agent 执行 Shell 命令，极大地扩展了 Agent 的能力边界。

```python
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "安装 pandas 库，然后检查版本号"
    }]
})

# Agent 内部调用：
# execute(command="pip install pandas")
# execute(command="python -c \"import pandas; print(pandas.__version__)\"")
```

### 沙箱与安全性

`execute` 工具带有沙箱机制，以防止 Agent 执行危险操作：

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")

# 创建 Agent 时可以通过配置影响 Shell 执行的安全边界
agent = create_deep_agent(llm)
```

::: warning 生产环境安全建议
- 在生产环境中始终启用沙箱限制
- 避免让 Agent 执行涉及网络、系统配置的敏感命令
- 对 Agent 的工作目录进行隔离，防止越权访问
- 审查 Agent 的执行日志，确保命令符合预期
:::

### 常见使用场景

```python
# 场景 1：运行测试
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "运行 pytest 并修复失败的测试用例"
    }]
})

# 场景 2：数据处理
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "用 Python 脚本处理 data.csv，统计每列的缺失值比例"
    }]
})

# 场景 3：项目构建
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "执行 npm run build，检查是否有构建错误"
    }]
})
```

## 子代理工具：task

`task` 工具允许 Agent 将子任务委派给独立的子 Agent 执行。子 Agent 拥有自己独立的对话上下文，完成后将结果汇报给主 Agent。

```python
# 当 Agent 面对多步骤复杂任务时，会使用 task 进行委派
result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "重构整个项目的日志系统：1)调研当前日志使用情况 2)设计新方案 3)逐文件替换"
    }]
})

# Agent 可能会：
# task("调研项目中所有日志相关的代码，列出使用方式和问题")
# task("将 src/api/routes.py 中的 print 语句替换为标准 logging 调用")
```

`task` 工具的详细机制和高级用法将在第 5 篇《子代理与任务委派》中深入讲解。

## 系统提示词如何教会模型使用工具

Deep Agents 的一个核心设计是：**系统提示词（System Prompt）本身就是工具使用的"教科书"**。当你调用 `create_deep_agent` 时，框架会自动注入一段精心编写的系统提示词，其中包含：

1. **工具使用策略**：何时该用哪个工具
2. **最佳实践**：例如"修改文件时用 edit_file 而非 write_file"
3. **规划指导**：面对复杂任务时先 write_todos
4. **错误处理**：工具执行失败后的重试策略

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")
agent = create_deep_agent(llm)

# 此时 agent 内部已经包含了完整的系统提示词
# 系统提示词教会 LLM：
# - 遇到文件操作任务 → 使用文件系统工具
# - 需要执行命令 → 使用 execute
# - 复杂任务 → 先 write_todos 做计划
# - 可独立子任务 → 使用 task 委派
```

这种"通过提示词教模型用工具"的方式是 Deep Agents 的核心上下文工程策略，它不需要额外的微调或训练，仅通过自然语言指令就能引导模型高效使用工具。

## 实战：文档整理与摘要 Agent

下面是一个综合运用多种内置工具的完整案例——构建一个能自动整理和摘要文档的 Agent。

### 准备测试数据

首先创建几个测试文档：

```python
import os

# 创建测试目录和文件
os.makedirs("./docs_demo", exist_ok=True)

files = {
    "meeting_0401.md": "# 4月1日会议纪要\n\n## 参会人员\n张三、李四、王五\n\n## 讨论内容\n1. Q1 销售数据回顾，总体达成率 95%\n2. Q2 目标制定，计划增长 20%\n3. 新产品线上线时间确定为 5 月中旬\n\n## 待办\n- 李四：整理 Q1 数据报告\n- 王五：准备新产品上线方案",
    "meeting_0408.md": "# 4月8日会议纪要\n\n## 参会人员\n张三、李四、赵六\n\n## 讨论内容\n1. Q1 数据报告已完成，发现华东区域增长最快\n2. 新产品原型评审通过\n3. 客户反馈系统需要升级\n\n## 待办\n- 张三：联系华东区域团队深入了解\n- 赵六：制定客户反馈系统升级方案",
    "meeting_0415.md": "# 4月15日会议纪要\n\n## 参会人员\n张三、王五、赵六\n\n## 讨论内容\n1. 华东区域成功经验：本地化运营 + 社群营销\n2. 新产品上线方案确认，分三个阶段发布\n3. 反馈系统升级预算批准\n\n## 待办\n- 王五：协调新产品第一阶段上线\n- 赵六：启动反馈系统升级开发",
}

for filename, content in files.items():
    with open(f"./docs_demo/{filename}", "w") as f:
        f.write(content)
```

### 运行 Agent

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv

load_dotenv()

llm = ChatOpenAI(model="gpt-4o")
agent = create_deep_agent(llm)

result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": """请帮我整理 ./docs_demo 目录下的所有会议纪要：
1. 先查看有哪些文件
2. 读取每份会议纪要
3. 提取所有待办事项，按负责人归类
4. 生成一份汇总报告，保存到 ./docs_demo/summary.md"""
    }]
})

# 打印最终回复
print(result["messages"][-1].content)
```

### Agent 执行流程

Agent 会按照如下流程自动完成任务：

```
[write_todos] 制定计划
  → [ls] 查看 ./docs_demo 目录
    → [glob] 匹配 *.md 文件
      → [read_file] 读取 meeting_0401.md
      → [read_file] 读取 meeting_0408.md
      → [read_file] 读取 meeting_0415.md
        → 分析提取待办事项
          → [write_file] 生成 summary.md
```

生成的 `summary.md` 大致内容：

```markdown
# 会议纪要汇总报告

## 会议概览
| 日期 | 主要议题 | 参会人数 |
|------|----------|----------|
| 4月1日 | Q1回顾与Q2目标 | 3人 |
| 4月8日 | 数据报告与产品评审 | 3人 |
| 4月15日 | 区域经验与上线方案 | 3人 |

## 待办事项（按负责人）

### 张三
- [ ] 联系华东区域团队深入了解

### 李四
- [ ] 整理 Q1 数据报告

### 王五
- [ ] 准备新产品上线方案
- [ ] 协调新产品第一阶段上线

### 赵六
- [ ] 制定客户反馈系统升级方案
- [ ] 启动反馈系统升级开发
```

## 核心概念回顾

| 工具 | 类别 | 用途 | 关键特点 |
|------|------|------|----------|
| `write_todos` | 规划 | 任务分解与进度追踪 | 无副作用的上下文工程策略 |
| `read_file` | 文件系统 | 读取文件内容 | 支持文本文件 |
| `write_file` | 文件系统 | 创建或覆盖文件 | 适合生成新文件 |
| `edit_file` | 文件系统 | 局部修改文件 | 比 write_file 更精细 |
| `ls` | 文件系统 | 列出目录内容 | 了解工作空间结构 |
| `glob` | 文件系统 | 模式匹配搜索文件 | 支持通配符和递归 |
| `grep` | 文件系统 | 搜索文件内容 | 支持正则表达式 |
| `execute` | Shell | 执行 Shell 命令 | 带沙箱安全机制 |
| `task` | 子代理 | 委派子任务 | 独立上下文执行 |
