# Deep Agents 基础：快速构建你的第一个 Agent

上一篇我们认识了深度 Agent 的概念并运行了 Hello World 示例。本篇将深入 `create_deep_agent()` 的配置方式，掌握流式输出、消息流机制和模型切换，最终构建一个实用的文件整理 Agent。

## create_deep_agent() 详解

`create_deep_agent()` 是 Deep Agents 框架的核心入口。它接受一个 LLM 实例和可选配置，返回一个**编译好的 LangGraph 图**，可以直接 `invoke()` 或 `stream()` 执行。

### 基础用法

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")
agent = create_deep_agent(llm)
```

这一行代码背后，框架帮你完成了：

1. 注入详细的系统提示词（数千 Token 的最佳实践指导）
2. 注册全套内置工具（规划、文件、Shell、子代理等）
3. 构建 ReAct 循环的图结构（Agent 节点 + Tools 节点 + 条件边）
4. 编译为可执行的 LangGraph 图

### 配置参数

`create_deep_agent()` 支持多个可选参数来定制 Agent 行为：

```python
agent = create_deep_agent(
    llm,                          # 必填：LLM 实例
    tools=[],                     # 可选：额外的自定义工具列表
    prompt="",                    # 可选：追加到系统提示词末尾的自定义指令
)
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `llm` | `BaseChatModel` | 支持 Tool Calling 的 LLM 实例 |
| `tools` | `list[Tool]` | 在内置工具基础上追加的自定义工具 |
| `prompt` | `str` | 追加到系统提示词末尾的额外指令 |

::: tip 返回值是 LangGraph 图
`create_deep_agent()` 返回的是一个标准的 LangGraph `CompiledGraph`。这意味着你可以使用 LangGraph 的所有特性：`invoke()`、`stream()`、检查点、可视化等。
:::

## 使用 invoke() 执行 Agent

`invoke()` 是最简单的执行方式——传入输入，等待 Agent 完成所有工作后返回最终结果。

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv

load_dotenv()

llm = ChatOpenAI(model="gpt-4o")
agent = create_deep_agent(llm)

result = agent.invoke({
    "messages": [
        {"role": "user", "content": "列出当前目录下的所有文件"}
    ]
})

# result 是一个字典，包含最终状态
print(result["messages"][-1].content)
```

`invoke()` 的输入和输出都是状态字典。输入时你只需要提供 `messages` 字段，Agent 执行完成后返回的状态中包含完整的消息历史。

### 理解返回结果

```python
result = agent.invoke({
    "messages": [{"role": "user", "content": "你好"}]
})

# 遍历所有消息查看完整对话过程
for msg in result["messages"]:
    print(f"[{msg.type}] {msg.content[:100]}")
```

返回的 `messages` 列表包含了完整的对话过程，你可以从中追溯 Agent 的每一步决策。

## 使用 stream() 实现流式输出

在实际应用中，深度 Agent 的执行可能持续几十秒甚至几分钟。使用 `stream()` 可以**实时观察 Agent 的执行过程**，而不必等待全部完成。

### 基础流式输出

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv

load_dotenv()

llm = ChatOpenAI(model="gpt-4o")
agent = create_deep_agent(llm)

# stream() 逐步返回每个节点的执行结果
for chunk in agent.stream(
    {"messages": [{"role": "user", "content": "创建一个 Python 项目结构"}]},
    stream_mode="updates",
):
    # chunk 是一个字典，key 是节点名称，value 是该节点的输出
    for node_name, node_output in chunk.items():
        print(f"\n--- {node_name} ---")
        if "messages" in node_output:
            for msg in node_output["messages"]:
                if hasattr(msg, "content") and msg.content:
                    print(msg.content[:200])
                if hasattr(msg, "tool_calls") and msg.tool_calls:
                    for tc in msg.tool_calls:
                        print(f"  [调用工具] {tc['name']}")
```

### stream_mode 说明

| 模式 | 说明 |
|------|------|
| `"updates"` | 每个节点执行完毕后返回该节点的输出（最常用） |
| `"values"` | 每个节点执行完毕后返回完整的当前状态 |

::: warning 注意
`stream()` 返回的是每个**图节点**执行后的结果，而不是 LLM 的逐 Token 流式输出。每个 chunk 对应一次"Agent 思考"或"工具执行"。
:::

## 理解消息流

深度 Agent 的执行本质上是一连串消息的交互。理解消息流对调试和定制 Agent 至关重要。

### 消息类型

```
HumanMessage    →  用户输入
AIMessage       →  模型回复（可能包含 tool_calls）
ToolMessage     →  工具执行结果
```

### 一个典型的消息流

当你要求 Agent "创建一个 hello.txt 文件"时，消息流大致如下：

```
1. HumanMessage:  "创建一个 hello.txt 文件，内容为 Hello World"
2. AIMessage:     [tool_calls: write_file(path="hello.txt", content="Hello World")]
3. ToolMessage:   "文件已成功写入: hello.txt"
4. AIMessage:     "我已经创建了 hello.txt 文件，内容为 Hello World。"
```

### 复杂任务的消息流

面对复杂任务，Agent 会进行多轮工具调用：

```
1. HumanMessage:  "分析当前目录下所有 Python 文件的代码行数"
2. AIMessage:     [tool_calls: write_todos(todos=["列出所有 .py 文件", "逐个统计行数", "汇总报告"])]
3. ToolMessage:   "Todo 列表已创建"
4. AIMessage:     [tool_calls: glob(pattern="**/*.py")]
5. ToolMessage:   "找到 3 个文件: main.py, utils.py, test_main.py"
6. AIMessage:     [tool_calls: execute(command="wc -l main.py utils.py test_main.py")]
7. ToolMessage:   "  42 main.py\n  18 utils.py\n  35 test_main.py\n  95 total"
8. AIMessage:     "分析完成！当前目录下共有 3 个 Python 文件，总计 95 行代码..."
```

注意第 2 步——Agent **先制定了计划**，然后按计划逐步执行。这就是深度 Agent 的规划能力。

### 查看完整消息流的工具函数

```python
def print_message_flow(result):
    """打印 Agent 执行的完整消息流"""
    for i, msg in enumerate(result["messages"]):
        prefix = f"[{i}] {msg.type:12s}"
        if hasattr(msg, "tool_calls") and msg.tool_calls:
            tools = ", ".join(tc["name"] for tc in msg.tool_calls)
            print(f"{prefix} -> 调用工具: {tools}")
        elif msg.content:
            content = msg.content[:80].replace("\n", " ")
            print(f"{prefix} {content}")
        else:
            print(f"{prefix} (空内容)")
```

## 使用 init_chat_model 切换模型

Deep Agents 是**模型无关**的，支持任何实现了 Tool Calling 的 LLM。LangChain 提供了 `init_chat_model()` 工具函数，可以用统一的方式创建不同提供商的模型实例。

### 安装对应的模型包

```bash
# OpenAI
pip install langchain-openai

# Anthropic
pip install langchain-anthropic

# Google
pip install langchain-google-genai
```

### 使用 init_chat_model

```python
from langchain.chat_models import init_chat_model

# OpenAI
llm = init_chat_model("gpt-4o", model_provider="openai")

# Anthropic
llm = init_chat_model("claude-sonnet-4-20250514", model_provider="anthropic")

# Google
llm = init_chat_model("gemini-2.0-flash", model_provider="google_genai")
```

### 用环境变量切换模型

一个实用的模式是通过环境变量控制使用哪个模型：

```python
import os
from langchain.chat_models import init_chat_model
from deepagents import create_deep_agent
from dotenv import load_dotenv

load_dotenv()

# 从环境变量读取模型配置
model_name = os.getenv("MODEL_NAME", "gpt-4o")
model_provider = os.getenv("MODEL_PROVIDER", "openai")

llm = init_chat_model(model_name, model_provider=model_provider)
agent = create_deep_agent(llm)
```

对应的 `.env` 文件：

```env
# 切换模型只需修改这两行
MODEL_NAME=gpt-4o
MODEL_PROVIDER=openai

OPENAI_API_KEY=sk-your-key
```

::: tip 选择合适的模型
深度 Agent 需要较强的指令遵循和工具调用能力。建议使用 GPT-4o、Claude Sonnet/Opus、Gemini 2.0 Flash 等较强的模型。较弱的模型可能无法正确使用复杂的工具组合。
:::

## 实战：构建文件整理 Agent

现在让我们综合运用本篇所学，构建一个实用的**文件整理 Agent**。它能够扫描指定目录，按文件类型自动分类整理。

### 完整代码

```python
"""
文件整理 Agent：扫描目录并按文件类型分类整理
"""
import os
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI
from dotenv import load_dotenv

load_dotenv()

def create_file_organizer():
    """创建一个文件整理 Agent"""
    llm = ChatOpenAI(model="gpt-4o")

    # 通过 prompt 参数给 Agent 注入领域知识
    agent = create_deep_agent(
        llm,
        prompt="""你是一个文件整理助手。整理文件时请遵循以下规则：
1. 先用 ls 查看目录内容，了解有哪些文件
2. 用 write_todos 制定整理计划
3. 按文件扩展名分类：
   - 图片：.jpg, .png, .gif, .svg → images/
   - 文档：.pdf, .doc, .docx, .txt → documents/
   - 代码：.py, .js, .ts, .html, .css → code/
   - 数据：.csv, .json, .xml → data/
   - 其他：归入 misc/
4. 使用 execute 工具执行 mkdir 和 mv 命令
5. 完成后输出整理报告""",
    )
    return agent


def main():
    agent = create_file_organizer()

    # 设定要整理的目录（这里用当前目录演示）
    target_dir = os.getcwd()

    # 使用 stream 实时观察 Agent 的工作过程
    print("文件整理 Agent 启动中...\n")

    for chunk in agent.stream(
        {
            "messages": [
                {
                    "role": "user",
                    "content": f"请整理 {target_dir} 目录下的文件，按文件类型分类到子文件夹中。注意：不要移动 .py 文件和 .env 文件，只整理其他类型的文件。",
                }
            ]
        },
        stream_mode="updates",
    ):
        for node_name, node_output in chunk.items():
            if node_name == "agent" and "messages" in node_output:
                for msg in node_output["messages"]:
                    if hasattr(msg, "tool_calls") and msg.tool_calls:
                        for tc in msg.tool_calls:
                            print(f"  [工具调用] {tc['name']}: {str(tc.get('args', ''))[:80]}")
                    elif msg.content:
                        print(f"\n{msg.content}")
            elif node_name == "tools" and "messages" in node_output:
                for msg in node_output["messages"]:
                    if msg.content:
                        preview = msg.content[:120].replace("\n", " ")
                        print(f"  [工具结果] {preview}")


if __name__ == "__main__":
    main()
```

### 运行效果

```bash
python file_organizer.py
```

Agent 的执行过程大致如下：

```
文件整理 Agent 启动中...

  [工具调用] ls: {'path': '/Users/you/project'}
  [工具结果] report.pdf  data.csv  photo.jpg  notes.txt  main.py  ...
  [工具调用] write_todos: {'todos': ['创建分类目录', '移动图片文件', '移动文档文件', ...]}
  [工具结果] Todo 列表已创建
  [工具调用] execute: {'command': 'mkdir -p images documents data misc'}
  [工具结果] (目录创建成功)
  [工具调用] execute: {'command': 'mv photo.jpg images/'}
  [工具结果] (移动成功)
  ...

整理完成！以下是整理报告：
- images/: 1 个文件 (photo.jpg)
- documents/: 2 个文件 (report.pdf, notes.txt)
- data/: 1 个文件 (data.csv)
- 保留未移动: main.py, .env
```

### 代码解析

这个示例展示了深度 Agent 的几个关键能力：

1. **自定义指令**：通过 `prompt` 参数注入文件分类规则
2. **规划能力**：Agent 先用 `write_todos` 制定计划，再逐步执行
3. **多工具协作**：`ls` 查看目录 → `write_todos` 制定计划 → `execute` 创建目录和移动文件
4. **流式输出**：使用 `stream()` 实时观察 Agent 的工作过程

## 核心概念回顾

| 概念 | 说明 |
|------|------|
| `create_deep_agent(llm)` | 创建深度 Agent，返回编译好的 LangGraph 图 |
| `agent.invoke()` | 同步执行，等待完成后返回最终状态 |
| `agent.stream()` | 流式执行，逐步返回每个节点的输出 |
| `stream_mode="updates"` | 每个节点执行后返回该节点的增量输出 |
| `HumanMessage` | 用户输入消息 |
| `AIMessage` | 模型回复，可能包含 `tool_calls` |
| `ToolMessage` | 工具执行结果 |
| `init_chat_model()` | 统一接口创建不同提供商的模型实例 |
| `prompt` 参数 | 向系统提示词追加自定义指令 |
