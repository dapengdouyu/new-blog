# Deep Agents 进阶：自定义工具与提示词

Deep Agents 的内置工具覆盖了通用场景，但实际业务中往往需要 Agent 具备领域特定的能力——查询数据库、调用内部 API、执行业务规则等。本篇将教你如何通过自定义工具和系统提示词，将 Deep Agents 打造为你领域中的专属助手。

## 为什么需要自定义？

内置工具解决的是"通用能力"问题，而业务场景需要"专业能力"：

```
通用 Agent                          领域 Agent
├── 读写文件 ✓                      ├── 读写文件 ✓
├── 执行命令 ✓                      ├── 执行命令 ✓
├── 任务规划 ✓                      ├── 任务规划 ✓
└── 搜索文件 ✓                      ├── 查询客户数据库 ← 自定义工具
                                    ├── 创建工单 ← 自定义工具
                                    ├── 计算退款金额 ← 自定义工具
                                    └── 客服人格与话术 ← 自定义提示词
```

Deep Agents 提供两个扩展入口：
- **`tools` 参数**：添加自定义工具
- **`system_prompt` 参数**：注入自定义系统提示词

## 创建自定义工具

### 使用 @tool 装饰器

Deep Agents 使用 LangChain 生态的 `@tool` 装饰器定义工具。模型通过函数的 **名称**、**docstring** 和 **类型注解** 来理解工具的用途和参数。

```python
from langchain_core.tools import tool

@tool
def get_weather(city: str) -> str:
    """查询指定城市的当前天气信息。当用户询问天气相关问题时使用此工具。

    Args:
        city: 城市名称，如"北京"、"上海"
    """
    # 实际场景中调用天气 API
    weather_data = {
        "北京": "晴，25°C，湿度 40%",
        "上海": "多云，28°C，湿度 65%",
        "广州": "小雨，30°C，湿度 80%",
    }
    return weather_data.get(city, f"暂无 {city} 的天气数据")
```

### 三要素：名称、描述、类型注解

LLM 是通过以下三个要素来决定何时、如何调用工具的：

```python
@tool
def search_products(
    keyword: str,
    category: str = "all",
    max_price: float = 0,
) -> str:
    """在商品数据库中搜索商品。当用户想查找或购买商品时使用。

    Args:
        keyword: 搜索关键词，如"手机壳"、"蓝牙耳机"
        category: 商品分类，可选值：all, electronics, clothing, food
        max_price: 最高价格限制，0 表示不限
    """
    # 模拟数据库查询
    return f"找到 3 件匹配 '{keyword}' 的商品（分类：{category}，价格上限：{max_price}）"
```

| 要素 | 来源 | 作用 |
|------|------|------|
| 工具名称 | 函数名 `search_products` | LLM 通过名称识别工具 |
| 工具描述 | docstring | LLM 判断何时使用此工具 |
| 参数定义 | 类型注解 + Args 说明 | LLM 知道传什么参数 |

::: warning docstring 是工具的灵魂
模型完全依赖 docstring 来理解工具的用途。模糊的描述会导致工具被错误调用或漏用。务必写清楚：**这个工具做什么、什么时候该用它、参数代表什么**。
:::

### 传递自定义工具给 Agent

使用 `create_deep_agent` 的 `tools` 参数注入自定义工具：

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from dotenv import load_dotenv

load_dotenv()

@tool
def query_order(order_id: str) -> str:
    """查询订单状态。当用户提供订单号并询问订单相关问题时使用。

    Args:
        order_id: 订单编号，格式如 ORD-20240401-001
    """
    orders = {
        "ORD-20240401-001": "已发货，预计明天到达",
        "ORD-20240401-002": "处理中，预计后天发货",
    }
    return orders.get(order_id, f"未找到订单 {order_id}")

@tool
def calculate_refund(order_id: str, reason: str) -> str:
    """计算退款金额。当用户申请退款时使用。

    Args:
        order_id: 订单编号
        reason: 退款原因，如"质量问题"、"不想要了"
    """
    # 模拟退款计算逻辑
    if reason == "质量问题":
        return f"订单 {order_id} 可全额退款：￥299.00"
    return f"订单 {order_id} 扣除运费后可退：￥279.00"

# 创建带自定义工具的 Agent
llm = ChatOpenAI(model="gpt-4o")
agent = create_deep_agent(
    llm,
    tools=[query_order, calculate_refund],  # 注入自定义工具
)

result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "帮我查一下订单 ORD-20240401-001 的状态"
    }]
})

print(result["messages"][-1].content)
```

自定义工具会与内置工具**共存**——Agent 既能使用你提供的业务工具，也保留了文件操作、Shell 执行等通用能力。

## 自定义系统提示词

### system_prompt 参数

`system_prompt` 参数让你可以为 Agent 注入额外的指令，定制它的行为风格、专业知识和限制条件。这段提示词会作为**自定义指令**插入到 Deep Agents 默认系统提示词中。

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")

agent = create_deep_agent(
    llm,
    system_prompt="""你是一位资深的 Python 代码审查专家。

你的职责：
1. 审查用户提供的 Python 代码
2. 指出潜在的 bug、性能问题和安全隐患
3. 给出具体的改进建议和修改后的代码

审查风格：
- 先肯定代码的优点，再指出问题
- 每个问题都要给出严重程度（高/中/低）
- 提供修改后的代码片段
- 使用中文进行沟通"""
)

result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "请审查 ./src/main.py 的代码质量"
    }]
})
```

### system_prompt 与默认提示词的关系

Deep Agents 有一套默认系统提示词，教模型如何使用内置工具。你的 `system_prompt` 是在默认提示词的基础上**追加**的，不会覆盖默认行为：

```
最终系统提示词 = Deep Agents 默认提示词（工具使用策略）
                + 你的 system_prompt（领域指令）
```

这意味着 Agent 既懂得如何使用工具，又具备你定义的专业知识和人格特征。

### 提示词设计建议

```python
# 好的 system_prompt 示例
system_prompt = """你是一位专业的数据分析师。

## 角色定位
- 擅长使用 Python 和 pandas 进行数据分析
- 能够生成可视化报告和数据洞察

## 工作规范
- 处理数据前先检查数据质量（缺失值、异常值）
- 分析结果要附带置信度说明
- 所有输出文件保存到 ./output 目录

## 限制
- 不要执行删除原始数据的操作
- 不要在代码中硬编码数据库密码"""
```

::: tip 好的 system_prompt 包含什么
1. **角色定位**：Agent 扮演什么角色，具备什么专业知识
2. **工作规范**：执行任务时应遵循的步骤和标准
3. **限制条件**：Agent 不能做什么，需要避免什么
4. **输出要求**：结果的格式、存放位置等
:::

## 两者结合：tools + system_prompt

最强大的用法是同时定制工具和提示词：

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from dotenv import load_dotenv

load_dotenv()

# ---------- 自定义工具 ----------

@tool
def query_customer(customer_id: str) -> str:
    """查询客户信息。当需要了解客户详情时使用。

    Args:
        customer_id: 客户编号，如 C10001
    """
    customers = {
        "C10001": "姓名：张三，等级：VIP，注册时间：2023-01-15，历史订单：42笔",
        "C10002": "姓名：李四，等级：普通，注册时间：2024-03-20，历史订单：3笔",
    }
    return customers.get(customer_id, f"未找到客户 {customer_id}")

@tool
def query_order(order_id: str) -> str:
    """查询订单详情。当用户询问订单相关问题时使用。

    Args:
        order_id: 订单编号
    """
    orders = {
        "ORD-2024-001": "商品：无线耳机，金额：￥299，状态：已发货，客户：C10001",
        "ORD-2024-002": "商品：手机壳，金额：￥39，状态：待发货，客户：C10001",
    }
    return orders.get(order_id, f"未找到订单 {order_id}")

@tool
def create_ticket(
    customer_id: str,
    issue_type: str,
    description: str,
    priority: str = "normal",
) -> str:
    """创建客服工单。当需要记录客户问题或升级处理时使用。

    Args:
        customer_id: 客户编号
        issue_type: 问题类型，可选值：refund, complaint, inquiry, technical
        description: 问题描述
        priority: 优先级，可选值：low, normal, high, urgent
    """
    return f"工单已创建：TK-{customer_id}-{issue_type}（优先级：{priority}）"

@tool
def calculate_refund(order_id: str, reason: str) -> str:
    """计算退款金额。当客户申请退款时使用。

    Args:
        order_id: 订单编号
        reason: 退款原因
    """
    refund_rules = {
        "质量问题": "全额退款",
        "7天无理由": "扣除运费后退款",
        "未收到货": "全额退款并补偿优惠券",
    }
    rule = refund_rules.get(reason, "需人工审核")
    return f"订单 {order_id} 退款方案：{rule}"

# ---------- 自定义提示词 ----------

CUSTOMER_SERVICE_PROMPT = """你是"小智"，一位专业、友善的在线客服。

## 服务准则
1. 始终保持礼貌和耐心，使用"您"而非"你"
2. 先确认客户身份（查询客户信息），再处理问题
3. 涉及退款时必须查询订单详情并计算退款金额
4. 无法解决的问题创建工单升级处理

## 话术规范
- 开场白：先问候客户，确认身份
- 处理中：告知客户正在查询，请稍等
- 结束语：确认问题是否解决，礼貌道别

## 权限限制
- 退款金额超过 500 元需创建工单由主管审批
- 不能修改客户个人信息
- 不能承诺不在规则范围内的补偿"""

# ---------- 创建 Agent ----------

llm = ChatOpenAI(model="gpt-4o")
agent = create_deep_agent(
    llm,
    tools=[query_customer, query_order, create_ticket, calculate_refund],
    system_prompt=CUSTOMER_SERVICE_PROMPT,
)

# ---------- 模拟客服对话 ----------

result = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "你好，我是客户 C10001，我的订单 ORD-2024-001 收到了但是耳机有质量问题，想申请退款"
    }]
})

print(result["messages"][-1].content)
```

Agent 的执行流程：

```
[query_customer] 查询 C10001 → VIP 客户，张三
  → [query_order] 查询 ORD-2024-001 → 无线耳机，￥299
    → [calculate_refund] 质量问题 → 全额退款 ￥299
      → 综合信息，按客服话术回复
```

预期回复类似：

```
您好张三女士/先生！我是客服小智，很高兴为您服务。

我已经为您查询到订单 ORD-2024-001（无线耳机，￥299）的信息。
非常抱歉给您带来了不好的体验！

关于您反馈的质量问题，根据我们的退款政策，您可以获得全额退款 ￥299.00。

退款将在 1-3 个工作日内原路退回您的支付账户。请问还有其他问题需要帮助吗？
```

## AGENTS.md：Harness Agent 配置

除了通过代码传入 `system_prompt`，Deep Agents 还支持使用 **AGENTS.md 文件** 来配置 Agent 的行为。这是 Deep Agents Harness（Agent 运行框架）的核心配置方式，特别适合部署场景。

### 什么是 AGENTS.md？

`AGENTS.md` 是一个 Markdown 文件，放在项目根目录下，定义 Agent 的身份、工作流程和行为规范。Deep Agents 的 CLI 和部署服务会自动读取它作为系统提示词的一部分。

```markdown
# 编码助手 Agent

你是一位经验丰富的软件工程师，帮助用户完成编码任务。你在沙箱环境中工作，拥有完整的 Shell 访问权限。

## 工作流程

每个任务请按以下流程执行：

### 第一阶段：规划
- 仔细阅读任务描述
- 使用 `glob` 和 `grep` 探索项目结构
- 使用 `write_todos` 制定实施计划
- 如果任务不清晰，先询问用户

### 第二阶段：实现
- 按计划逐步执行
- 每次修改后运行测试
- 测试失败时先调试再继续
- 更新 todo 状态

### 第三阶段：验证
- 运行完整测试套件：`execute("python -m pytest")`
- 运行代码检查：`execute("ruff check .")`
- 逐一审查修改的文件
- 确认修改确实解决了原始问题

## 编码规范

- 保持与现有代码风格一致
- 为新功能编写测试
- 修改应最小化，不改动无关代码

## 子代理使用

对于复杂任务，委派给子代理：
- 使用 `task(subagent_type="researcher")` 做调研
- 使用 `task(subagent_type="general-purpose")` 执行独立子任务
```

### AGENTS.md 与 system_prompt 的区别

| 特性 | AGENTS.md | system_prompt 参数 |
|------|-----------|-------------------|
| 配置方式 | Markdown 文件 | Python 代码 |
| 适用场景 | CLI 和部署 | SDK 编程调用 |
| 版本控制 | 自然纳入 Git | 需要在代码中管理 |
| 灵活性 | 静态文本 | 支持动态生成 |
| 子代理配置 | 支持子目录 AGENTS.md | 通过 subagents 参数 |

### 子代理的 AGENTS.md

子代理也可以有自己的 AGENTS.md，放在 `subagents/` 子目录下：

```
my-agent/
├── AGENTS.md                          # 主 Agent 配置
├── skills/                            # 技能目录
│   └── code-review/
│       └── SKILL.md
└── subagents/                         # 子代理配置
    └── market-researcher/
        ├── AGENTS.md                  # 子代理自己的行为定义
        └── skills/
            └── analyze-market/
                └── SKILL.md
```

### 通过 memory 参数加载 AGENTS.md

在 SDK 中，你可以通过 `memory` 参数将 AGENTS.md 加载到 Agent 中：

```python
from deepagents import create_deep_agent

agent = create_deep_agent(
    model="openai:gpt-4o",
    memory=["/memory/AGENTS.md"],  # 加载 AGENTS.md 到系统提示词
)
```

## Skills：技能系统

Deep Agents 支持 **Skills（技能）** 系统——一种模块化的能力扩展机制。每个 Skill 是一个包含 `SKILL.md` 的目录，定义了特定领域的工作流程和最佳实践。

### 什么是 Skill？

Skill 是一个标准化的知识模块，遵循 [Agent Skills 规范](https://agentskills.io/specification)。它包含：

- **SKILL.md**：定义技能的名称、描述、使用时机和具体步骤
- **辅助文件**（可选）：脚本、配置文件、模板等

```
skills/
├── web-research/
│   └── SKILL.md           # 网络研究技能
├── code-review/
│   ├── SKILL.md           # 代码审查技能
│   └── lint_check.py      # 辅助脚本
└── blog-post/
    └── SKILL.md           # 博客写作技能
```

### SKILL.md 的格式

每个 SKILL.md 使用 YAML frontmatter 定义元数据，后面跟 Markdown 格式的详细指令：

```markdown
---
name: code-review
description: 对代码变更进行结构化审查，检查正确性、风格、测试和潜在问题。
---

# 代码审查技能

在完成代码修改后使用此技能来验证你的工作。

## 审查检查清单

### 1. 正确性
- [ ] 修改解决了原始问题
- [ ] 没有对现有功能产生副作用
- [ ] 边界情况已处理
- [ ] 错误处理恰当

### 2. 代码质量
- [ ] 代码风格与现有代码一致
- [ ] 没有不必要的复杂度
- [ ] 变量和函数命名清晰

### 3. 测试
- [ ] 新功能有测试覆盖
- [ ] 现有测试仍然通过

## 执行步骤

1. 完整阅读每个修改的文件
2. 运行测试套件：`execute("python -m pytest -v")`
3. 运行代码检查：`execute("python /skills/code-review/lint_check.py .")`
4. 对照检查清单逐项验证
5. 发现问题则修复后重新审查
```

### SKILL.md 的元数据字段

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | 是 | 技能标识符，小写字母和连字符，如 `web-research` |
| `description` | 是 | 技能描述，用于 Agent 判断何时使用 |
| `license` | 否 | 许可证 |
| `compatibility` | 否 | 环境要求说明 |
| `allowed-tools` | 否 | 推荐使用的工具列表 |

### 在 Agent 中使用 Skills

通过 `skills` 参数指定技能目录路径：

```python
from deepagents import create_deep_agent

agent = create_deep_agent(
    model="openai:gpt-4o",
    skills=[
        "/skills/user/",      # 用户级技能
        "/skills/project/",   # 项目级技能（更高优先级）
    ],
)
```

::: tip 渐进式披露（Progressive Disclosure）
Skills 使用**渐进式披露**模式：Agent 启动时只看到技能的名称和描述摘要（写在系统提示词中）。当 Agent 判断某个技能适用于当前任务时，它会主动调用 `read_file` 读取完整的 SKILL.md 内容，获取详细步骤。这样既节省了上下文窗口，又保证 Agent 在需要时能获取完整指导。
:::

### 技能的优先级

当多个 source 中存在同名技能时，**后加载的覆盖先加载的**（last one wins）。这允许你分层管理技能：

```python
skills=[
    "/skills/base/",       # 基础技能（兜底）
    "/skills/user/",       # 用户自定义（覆盖基础）
    "/skills/project/",    # 项目特定（最高优先级）
]
```

### 创建自定义 Skill

以下是一个完整的自定义技能示例——数据分析技能：

**目录结构：**

```
skills/
└── data-analysis/
    ├── SKILL.md
    └── templates/
        └── report_template.md
```

**SKILL.md：**

```markdown
---
name: data-analysis
description: 使用 Python 和 pandas 进行结构化数据分析，生成洞察报告。当用户要求分析 CSV、Excel 数据或生成数据报告时使用。
---

# 数据分析技能

## 使用时机
- 用户要求分析数据文件（CSV、Excel、JSON）
- 需要生成数据洞察或统计报告
- 需要数据可视化

## 分析流程

### 第一步：数据探索
1. 使用 `read_file` 查看数据文件前几行
2. 确认列名、数据类型、行数
3. 使用 `write_todos` 制定分析计划

### 第二步：数据清洗
1. 检查缺失值
2. 检查异常值
3. 将结果记录到 `notes/data_quality.md`

### 第三步：分析执行
1. 按计划执行分析
2. 每完成一步记录中间结果

### 第四步：报告生成
1. 读取报告模板：`read_file("/skills/data-analysis/templates/report_template.md")`
2. 填充分析结果
3. 保存到 `output/report.md`

## 注意事项
- 大文件使用分块读取
- 保留原始数据，不做修改
- 数值结果保留 2 位小数
```

## MCP 集成

除了使用 `@tool` 自定义工具，Deep Agents 还支持通过 **MCP（Model Context Protocol）** 接入外部工具服务器。MCP 是一种标准化协议，允许 Agent 动态发现和调用外部服务提供的工具。

### 使用 langchain-mcp-adapters

```python
from deepagents import create_deep_agent
from langchain_openai import ChatOpenAI
from langchain_mcp_adapters.client import MultiServerMCPClient
from dotenv import load_dotenv

load_dotenv()

async def main():
    # 连接 MCP 服务器
    async with MultiServerMCPClient(
        {
            "filesystem": {
                "command": "npx",
                "args": [
                    "-y",
                    "@modelcontextprotocol/server-filesystem",
                    "/path/to/allowed/directory",
                ],
            },
            "database": {
                "command": "npx",
                "args": [
                    "-y",
                    "@modelcontextprotocol/server-sqlite",
                    "--db-path",
                    "./my_database.db",
                ],
            },
        }
    ) as mcp_client:
        # 获取 MCP 工具列表
        mcp_tools = mcp_client.get_tools()

        # 将 MCP 工具传递给 Agent
        llm = ChatOpenAI(model="gpt-4o")
        agent = create_deep_agent(
            llm,
            tools=mcp_tools,  # MCP 工具与内置工具共存
        )

        result = await agent.ainvoke({
            "messages": [{
                "role": "user",
                "content": "查询数据库中最近 10 条订单记录"
            }]
        })

        print(result["messages"][-1].content)

import asyncio
asyncio.run(main())
```

::: tip MCP 的优势
MCP 工具由外部服务器提供，可以跨语言、跨平台复用。同一个 MCP 服务器可以同时服务于多个 Agent 或应用，便于工具的集中管理和版本控制。
:::

## 工具设计最佳实践

### 1. 清晰的工具描述

```python
# 不好的示例
@tool
def query(id: str) -> str:
    """查询数据"""
    ...

# 好的示例
@tool
def query_customer_by_id(customer_id: str) -> str:
    """根据客户编号查询客户详细信息，包括姓名、等级、注册时间和历史订单数。
    当需要了解客户身份、验证客户信息或查看客户等级时使用此工具。

    Args:
        customer_id: 客户编号，格式为 C 加 5 位数字，如 C10001
    """
    ...
```

### 2. 合理的参数粒度

```python
# 不好的示例：参数过于宽泛
@tool
def process_data(data: str) -> str:
    """处理数据"""
    ...

# 好的示例：参数明确且有默认值
@tool
def analyze_csv(
    file_path: str,
    columns: str = "all",
    operation: str = "describe",
) -> str:
    """分析 CSV 文件的数据。支持描述性统计、缺失值检测和相关性分析。

    Args:
        file_path: CSV 文件路径
        columns: 要分析的列名，多列用逗号分隔，"all" 表示所有列
        operation: 分析类型，可选值：describe(描述统计), missing(缺失值), correlation(相关性)
    """
    ...
```

### 3. 返回结构化结果

```python
# 不好的示例：返回模糊的文本
@tool
def check_inventory(product_id: str) -> str:
    """检查库存"""
    return "有库存"

# 好的示例：返回结构化信息
@tool
def check_inventory(product_id: str) -> str:
    """检查指定商品的库存情况。

    Args:
        product_id: 商品编号
    """
    # 模拟库存查询
    inventory = {"P001": 150, "P002": 0, "P003": 23}
    stock = inventory.get(product_id, -1)
    if stock < 0:
        return f"商品 {product_id} 不存在"
    if stock == 0:
        return f"商品 {product_id} 已售罄（库存：0）"
    return f"商品 {product_id} 有货（库存：{stock} 件）"
```

### 4. 妥善处理错误

```python
@tool
def call_external_api(endpoint: str, params: str = "{}") -> str:
    """调用外部 API 获取数据。

    Args:
        endpoint: API 端点路径
        params: JSON 格式的请求参数
    """
    import json
    import requests

    try:
        parsed_params = json.loads(params)
    except json.JSONDecodeError:
        return "错误：params 不是有效的 JSON 格式"

    try:
        response = requests.get(
            f"https://api.example.com/{endpoint}",
            params=parsed_params,
            timeout=10,
        )
        response.raise_for_status()
        return response.text
    except requests.Timeout:
        return f"错误：API 请求超时（端点：{endpoint}）"
    except requests.RequestException as e:
        return f"错误：API 请求失败 — {str(e)}"
```

::: warning 工具函数不要抛出异常
工具函数应该捕获所有异常并返回错误描述字符串，而不是让异常向上抛出。这样 Agent 可以理解错误原因并决定下一步操作（重试、换方案或告知用户）。
:::

## 核心概念回顾

| 概念 | 说明 |
|------|------|
| `@tool` 装饰器 | 将 Python 函数声明为 Agent 可调用的工具 |
| `tools` 参数 | 向 `create_deep_agent` 注入自定义工具列表 |
| `system_prompt` 参数 | 注入自定义指令，定制 Agent 的行为和人格 |
| docstring | 工具的"说明书"，LLM 通过它理解工具用途 |
| 类型注解 | 定义工具参数的类型和结构 |
| AGENTS.md | Harness Agent 配置文件，定义 Agent 身份和工作流程 |
| `memory` 参数 | 加载 AGENTS.md 等记忆文件到系统提示词 |
| SKILL.md | 技能定义文件，遵循 Agent Skills 规范 |
| `skills` 参数 | 指定技能目录路径，启用技能系统 |
| 渐进式披露 | Skills 先展示摘要，Agent 按需读取完整内容 |
| MCP | Model Context Protocol，标准化的外部工具接入协议 |
| `langchain-mcp-adapters` | LangChain 生态的 MCP 客户端适配器 |
